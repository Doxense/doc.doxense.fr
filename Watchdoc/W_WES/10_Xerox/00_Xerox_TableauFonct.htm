Authentification : moyens dont dispose l'utilisateur pour s'authentifier sur le périphérique
Interface permettant l'authentification de l'utilisateur : interface Xerox interface Xerox
Authentification par Code PUK Puk = Print User Key. Dans Watchdoc, il s'agit d'un code associé à un compte utilisateur pour permettre à ce dernier de s'authentifier dans un WES. Le code PUK est généré par un algorithme. L'utilisateur peut le consulter dans la page "Mon compte" de Watchdoc. Par souci de sécurité, nous déconseillons l'utilisation du code PUK et recommandons l'utilisation d'un login (compte utilisateur)/code PIN. ✔ ✔

Authentification par Login/mot de passe

 ✔ ✔

Authentification par Login/Code PIN Code PIN (Personal Identification Number) est un code comportant au moins 4 chiffres. Il est par exemple utilisé sur un téléphone mobile ou un smartphone équipés d'une carte SIM. Le terme de code PIN n'est pas exclusif à la téléphonie mobile, on parle également de code PIN pour le code bancaire ou tout autre numéro d'identification personnel composé de 4 chiffres ou plus.

Codes analytiques Le code analytique est un code correspondant à un compte spécifique sur lequel les impressions demandées par les utilisateurs peuvent être imputées. Ce code permet donc la ventilation des coûts d'impression facilitant une comptabilité analytique des dépenses liées à l'impression.

Mode Haute Disponibilité (ou HA pour High Availability). Qualité d'un système, ou d'une partie d'un système (ex. : un périphérique), dont le pourcentage de temps durant lequel on peut l'utiliser est très élevé par rapport au temps global où il est en demande. (Source : OQLF) La haute disponibilité de Watchdoc repose sur la surveillance du Service Watchdoc et du spooler MS Windows ainsi que sur la sécurisation de la connectivité entre le poste de travail et le serveur d'impression et entre le périphérique d'impression et le serveur d'impression.

 ✔ ✔

WES compatible avec le mode multiserveur Ce mode permet d'adjoindre au serveur d'impression principal un ou des serveurs de secours capables de traiter les requêtes en cas de panne du serveur principal. Ce mode n'est pas compatible avec tous les modèles de périphériques (fonctionne avec les WES développés en Java). Il doit être configuré sur le profil de WES appliqué sur le périphérique.

 ✖ ✖

WES compatible avec le mode tolérance de pannes Mode rendant le WES compatible avec un équipement réseau qui gère la répartition de charge (load-balancing). Le WES, paramétré avec l'équipement qui gère le load-balancing, ne connaît pas précisément les serveurs d'impressions. Fonction compatible avec les WES développé en Java.

 ✔ ✔

WES apte à fonctionner en mode hors-ligne Mode rendant le périphérique d'impression capable de fonctionner de manière dégradée dans le cas où les serveurs d'impression (principal et de secours) sont en panne. En mode hors-ligne, l'impression est impossible, mais les autres fonctions du périphérique peuvent être proposées : photocopie, fax, scan. Si le mode hors-ligne est désactivé, en cas de panne des serveurs, toutes les fonctions du périphérique sont bloquées jusqu'à dépannage du serveur. Ce mode est configuré dans le profil WES appliqué sur le périphérique.
