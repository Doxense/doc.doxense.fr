WRS - Alimenter la table ASSETS

Principe

La table MS SQL® ASSETS permet d’afficher dans vos rapports des données non-présentes dans la base de données Watchdocstats (stockée dans la config.xml). Ces données peuvent provenir :

  • de la Console de supervision Wathcdoc (WSC) ;

  • du serveur IIS, au moyen d'une requête en ligne de commande.

Prérequis

Pour importer des données depuis la Console de supervision Watchdoc® (WSC), il est nécessaire :

  • disposer d'une licence WSC;

  • utiliser l'outil WCLI.

Procédure d'import depuis la Console de Supervision (WSC)

Pour importer des données de WSC :

  1. accédez à la Console de Supervision en tant qu'administrateur;

  2. cliquez sur le bouton Configuration avancée puis Service:

     

  3. Dans l'interface Configuration Système > section Envoi automatique des informations des périphériques dans la base de données :

    • cochez la case Activation ;

    • dans le champ de saisie, définissez la Fréquence d'envoi des données (en format CRON table ou à l'aide de l'assistant de configuration).

Nous vous recommandons de programmer un import hebdomadaire tenant compte des périodes de redémarrage de vos serveurs.

Procédure d'import des données IIS

L'outil Watchdoc Command Line (WCLI) est disponible dans le dossier d'installation de Watchdoc folder. Une Fiche pratique Utiliser WCLI est disponible dans Connect.

Pour importer des données avec l'outil WCLI :

  1. accédez à C:\Program Files\Doxense\Watchdoc\wcli.exe ;

  2. dans l'outil WCLI, lancez la commande /export ;

  3. les paramètres disponibles pour cette commande sont les suivants :

     

/server indiquer le nom du serveur /ServerWebPath Renseigner l’emplacement du fichier server.xml s’il n’est pas par défaut C:\inetpub\wwwroot\watchdoc\includes\servers.xml
/port indiquer le n° de port (s'il est spécifique) /configpath Renseigner l’emplacement du fichier server.xml s’il n’est pas par défaut c:\program files\doxense\Watchdoc\data\config.xml
/Password Renseigner le mot de passe Watchdoc

→ cette commande analyse le contenu du fichier Servers.xml sur le serveur IIS et collecte toutes les données attendues par la table Assets.

 

Exemple : le serveur IIS (SRVIIS) est déporté, un partage est créé avec le compte de service chargé d'exécuter la tâche planifiée : \\SRVIIS\WDSERVERS), avec le mot de passe de maintenance Watchdoc123 ;

La ligne de commande est la suivante :

”c:\program files\doxense\Watchdoc\wcli.exe” EXPORT /server=localhost/ServerWebPath=”\\SRVIIS\WDSERVER\servers.xml /password=Watchdoc123
