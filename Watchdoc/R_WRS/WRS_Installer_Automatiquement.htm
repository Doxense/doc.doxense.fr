WRS - Installer automatiquement Watchdoc for Reporting Services

A partir de MS SQL® 2012 R2 , il est possible de déployer automatiquement les rapports WRS à l'aide de l'utilitaire WRSTools.exe mis à votre disposition dans le package d'installation.

Accéder à l'outil WRSTOOLS.exe

Pour installer automatiquement WRS :

  1. depuis Connect, téléchargez le package d'installation WRS ;

  2. enregistrez le dossier d'archive sur votre serveur ;

  3. décompressez l'archive après voir vérifié qu'elle n'était pas bloquée ;

  4. parmi les fichiers de l'archive, repérez le fichier WRSTools.exe.


Avec la version MS SQL Express®, l'outil WRS® doit être installé sur le même serveur que celui qui héberge la base de données.Les instances créées avec SQL Express ne peuvent dépasser les 10Go. Pour pouvoir gérer plus d’informations, nous conseillons la création d’une base de données standard (ou une base de consolidation, elle-aussi standard, dans un domaine incluant un serveur master et d'autres serveurs).

Avec les autres version de MS SQL, l'outil WRS® peut être installé sur un autre serveur que celui qui héberge la base de données.

Installer automatiquement WRS®

Pour installer WRS® à l'aide de WRSTools :

  1. dans le dossier d'archive contenant le package d’installation WRS, cliquez sur le fichier WRSTools.exe ;

  2. vous accédez à l'interface d'installation WRS Tools constituée de deux onglets :

    • Settings ;

    • Advanced :


       

  1. sous l'onglet Settings, complétez les champs suivants :

    • Watchdoc server : indiquez le nom de votre server Watchdoc (Nom NETBIOS ou adresse IP) ;

    • Watchdoc root password : indiquez le mot de passe de maintenance Watchdoc ;

    • SQL Report server : indiquez le nom (NETBIOS ou adresse IP) de votre serveur SQL ;

    • WRS Licence File : indiquez le chemin d'accès à la licence Watchdoc for Reporting Services. Fourni par Doxense, ce fichier porte l'extension .wlk ;

    • Language : dans la liste, sélectionnez la langue de l'interface WRS :

WRS nécessite une licence qui lui est propre, différente de celle qui est fournie pour Watchdoc. Lors de la demande de licence, vous devrez fournir l'adresse MAC du serveur depuis lequel sera exécuté l'outil WRS.
Le compte utilisé par Watchdoc doit avoir le statut de propriétaire .dbo sur la base de données Watchdocstats afin pouvoir créer une base de données SQL et une procédure sauvegardée d'optimisation.

 

  1. Sous l'onglet Advanced, remplissez les champs suivants :

    • Watchdoc Port : s'il ne s'agit pas du port par défaut 5744, indiquez le numéro de port utilisé pour accéder à Watchdoc ;

    • Report server Login* : indiquez le nom d’utilisateur ajouté dans les droits Reporting Services (cf. partie précédente) ;

    • Report server password* : indiquez le mot de passe du compte utilisé (droits Reporting Services nécessaires) ;

    • Folder with *.rdl : indiquez dans ce champ l’emplacement du dossier comportant les fichiers .rdl (Reports, par défaut) ;

    • SQL Instance : indiquez le nom de l'instance installée propre à Watchdoc (Watchdoc, par défaut) ;

       

Pour compléter les champs de type Report :
- connectez-vous sur l'interface Web de Reporting Services :
(exemple : http://serverreportingservice/Reports_INSTANCESQL) ;
- si vous arrivez à vous connecter, cela signifie que votre compte est autorisé à attribuer des rôles de SQL Reporting Services ;
- sinon, demandez l’ajout de votre compte dans l’attribution des rôles de SQL Reporting Services.

 

  1. Une fois les paramètres remplis et vérifiés, cliquez sur le bouton Launch ( pour exécuter WRS ;

  1. Dans la boîte SQL Password not found saisissez le mot de passe du compte administrant l'instance SQL indiquée précédemment sous l'onglet Advanced, puis cliquez sur le bouton OK :


     

→ un message vous informe que l'installation d WRS est terminée :

 

  1. Cliquez sur le lien URL affiché pour accéder à l'interface SQL Server Reporting Services :



Déployer les rapports WRS

Dans le gestionnaire de rapports SQL Server Reporting Services, un dossier Watchdoc for Reporting Service s'affiche :

 

  1. Parmi les fichiers contenus dans ce dossier, cliquez sur l'entrée Statement pour accéder au rapport :


Si des problèmes surviennent lors de l'exécution de Statement, vous pouvez consulter les logs de WRS (depuis le dossier contenant WRSTools.exe).
