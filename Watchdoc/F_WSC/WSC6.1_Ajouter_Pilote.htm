Console de Supervision (WSC v6.1) - Ajouter un pilote (driver)

Prérequis

WSC v. 6.1 inclut un module de sécurité pour les pilotes d'impression. A partir de cette version, seuls les pilotes certifiés par Microsoft® avec une signature WHQL Signature numérique attribuée aux packages de pilotes répondant aux exigences des tests HLK (Windows Hardware Lab Kit) par Microsoft pour garantir la sécurité du pilote. (Source : https://learn.microsoft.com) valide peuvent être installés sur la console de supervision. Une exception existe pour les pilotes Epson® et Ricoh®.

 

Pour préparer les packages des pilotes à télécharger dans WSC v 6.1, il est nécessaire :

  • pour tous les pilotes (hormis Epson et Ricoh) :

    • d'utiliser l'outil DriverPackageExtractor.exe fourni par Doxense dans le package d'installation Watchdoc.

    • de disposer de l'outil de compression 7-Zip®(cf.https://7-zip.org/). 

      Si vous n'avez pas l'autorisation d'installer 7-Zip sur le serveur WSC, préparez le package dans un autre espace de travail. Sur cet espace, installez 7-Zip ainsi que les outils suivants, disponibles dans le package d'installation de Watchdoc :

      • DriverPackageExtractor.exe
      • DriverPackageExtractor.exe.config
      • CliWrap.dll
      • Doxense.Core.dll
      • Doxense.IO.Compression.dll
      • Doxense.Watchdoc.SpoolMonitor.dll


  • pour les pilotes Epson et Ricoh, il est nécessaire de les faire signer au préalable par Doxense, en vous adressant au Support via Connect.

     

Mise à jour : si vous effectuez une mise à jour à partir d'une version antérieure à la v. 6.1, vous devez désinstaller les pilotes existants et les recréer à l'aide de la procédure décrite ci-dessous.
Si vous n'avez pas mis à jour les pilotes mais que vous ajoutez un serveur dans le domaine ou un poste utilisateur sur lequel vous installez la dernière version de WPC for Windows, vous devrez également désinstaller les pilotes existants et les réinstaller à l'aide de DriverPackageExtractor.

Procédure

Télécharger le package certifié (tous les pilotes sauf Epson et Ricoh)

  1. En tant qu'administrateur, accédez au serveur qui héberge WSC.

  1. Via un navigateur, rendez-vous sur le site officiel du fabricant.

  2. Téléchargez le package du pilote correspondant au modèle du périphérique d'impression à installer : généralement, il s'agit d'une archive /fichier compressé.

  3. S'il ne s'agit pas d'un fichier compressé (un .exe, par exemple), compressez-le car WSC traite exclusivement des pilotes présentés au format .zip.

  4. Enregistrez le package sur le serveur.

Préparer les pilotes

Tous les pilotes sauf Epson et Ricoh

  1. Avec l'explorateur, rendez-vous dans le dossier C:\Program Files\Doxense\Watchdoc.

  2. Lancez l'outil DriverPackageExtractor.exe en tant qu'administrateur :

     

  3. Dans l'outil DriverPackageExtractor :

    • Path to 7Zip : indiquez le chemin d'accès à l'utilitaire 7-Zip ;

    • Driver archive : indiquez le chemin d'accès vers le fichier compressé du pilote (téléchargé précédemment) ;

    • Temporary folder : indiquez le chemin vers un dossier dans lequel sont enregistrés temporairement les fichiers extraits du package du pilote et dans le champ File name, attribuez un nom à ce fichier final. 

  4. Cliquez sur le bouton Prepare package and analyse.

  5. Indiquez l'endroit où vous souhaitez enregistrer le package du pilote une fois analysé par DriverPackageExtractor ;

  6. cliquez sur Save :

    -


  7. Lorsque le package est prêt, cliquez sur OK dans l'interface informant du succès de l'opération..

     

    è Dès lors, ce package est conforme aux attente de WSC et vous pouvez l'y ajouter.
    Si la préparation prend beaucoup de temps ou s'achève par un message d'erreur, il se peut que ce soit à cause du poids de l'archive. Dans ce cas allégez-le (cf. Alléger le dossier d'archive).

Pilotes Epson et Ricoh

  1. Via un navigateur, rendez-vous sur le site officiel du fabricant.

  2. Téléchargez le pilote correspondant aux périphériques d'impression à installer.

  3. Envoyez les packages des pilotes téléchargés au Support Doxense via Connect en demandant qu'ils soient signés.

  4. Enregistrez le package fourni en retour par Doxense sur le serveur qui héberge WSC.

Ajouter un pilote

  1. Accédez à WSC en tant qu'administrateur. 

  2. Depuis le Menu principal ,cliquez sur le menu Plug and Print> Pilotes :

  3. Depuis l'interface Pilotes, dans le menu du haut à droite, cliquez sur Ajouter un pilote.

  4. Dans la fenêtre Ajouter un pilote,

    • glissez le package du pilote (fichier compressé zip) depuis votre espace de travail ;

    • ou cliquez dans le cadre pour ouvrir l'explorateur de l'espace de travail et sélectionner le package du pilote à télécharger :


       

  1. Cliquez sur le bouton Suivant ;

  2. Dans l'interface Ajouter un pilote d'impression, quand le téléchargement s'est bien passé et que le package est valide, les champs suivants sont complétés :

    • Chemin vers .inf : chemin vers le fichier d'installation (.inf) présent dans le package du pilote et dans lequel se trouve la valeur DriverName.

    • Pilote : nom du pilote ou des pilotes contenus dans le package d'installation. S'il y en a plusieurs, choisissez dans la liste celui qui est compatible avec le(s) périphérique(s) utilisé(s).

    • Version : version du pilote.

    • Systèmes : version du système sur lequel est installé le pilote.

    • Package signé par : nom de l'émetteur (issuer) de la signature du pilote (Microsoft dans le cas d'un pilote WHQL ou Doxense, dans le cas de pilotes Epson ou Ricoh) :

       

  1. cliquez sur le bouton Suivant ;

  1. dans la fenêtre Créer une file universelle,

    1. cochez la case Créer si vous souhaitez créer une file universelle File logique créée exclusivement pour la Print API. Elle est créée automatiquement par défaut et nommée "Impression Sécurisée" lors de l'installation de Watchdoc. Elle est préconfigurée pour rediriger les fichiers de spool vers le pool d'impression global (#global). Cette file ne peut pas être supprimée. Telle qu'elle est configurée à l'orignie, elle ne peut pas servir à WPC for Windows. Pour que WPC for Windows puisse l'utiliser, il convient de la configurer (cf. Configuration de la file universelle). ;

    2. Nom : attribuez un nom à cette file. 

  2. Cliquez sur Valider.
    è Si l'opération s'est déroulée avec succès, le message suivant s'affiche. 

     


    èDans le cas contraire, un message d'erreur s'affiche. Cliquez sur le bouton pour afficher les logs dans lesquels sont listées les erreurs.
    Si vous n'arrivez pas à comprendre l'erreur et à réitérer l'opération avec succès, joingnez ces logs à un ticket Connect adressé au Support Doxense.

Créer une file universelle

Dans Watchdoc, la file universelle sert exclusivement à Watchdoc Print Client (cf. File universelle). Si vous souhaitez que le pilote serve à générer une file universelle,

  1. cochez la case Créer une file universelle;

  2. saisissez le Nom attribué à cette file ;

  3. cliquez sur Valider :


Associer une configuration

Lors de l'ajout du pilote, il est possible d'y ajouter une (ou plusieurs) configuration(s) de file d'impression.

Cela permet d'associer le pilote à une (ou plusieurs) configuration existante afin de gagner du temps ensuite lors de l'installation des files. En effet, il est alors possible d'installer les files sans avoir à les configurer. Outre le gain de temps, cette association garantit l'harmonisation des configurations lors de l'ajout de files en masse.

 

Si vous disposez d'un fichier de configuration modèle (.dat) disponible dans votre espace de travail, procédez directement à son ajout. 
Si vous ne disposez d'aucun fichier de configuration, procédez d'abord au téléchargement d'un fichier de configuration modèle avant de l'associer à votre pilote (cf. Télécharger un fichier de configuration de file (.dat)).

 

Si vous ne disposez pas de fichier de configuration (.dat), procédez comme suit :

  1. depuis l'interface Créez une file universelle avec ce pilote, cliquez sur le bouton Avancé ;

  2. depuis l'interface Associez des configurations à ce pilote, cliquez sur le bouton Ajouter ;

  3. parcourez votre espace de travail pour y sélectionner le fichier de configuration téléchargé ;

  4. réitérez l'opération autant de fois que de fichiers de configuration à ajouter (jusqu'à 10) ;

  5. cliquez sur Valider pour finaliser l'ajout du pilote :

Procédure spécifique à Xerox

La configuration du pilote Xerox n'est pas standard, Xerox fournissant un fichier .xml propre à la politique d'impression du client. Lors de l'ajout du fichier de configuration pour un périphérique Xerox, il convient donc de sélectionner le fichier CommonConfiguration.xml fourni par Xerox.

Ce fichier doit avoir été paramétré par nos soins préalablement, un fichier correspondant à une configuration de file d'impression.

 

→ Une fois le pilote configuré, vous pouvez configurer le modèle de périphérique.

Dépannages

Alléger un pilote

Le poids d'un fichier contenant le pilote varie entre 10 et 30 Mb. Si le fichier est conforme mais trop lourd, la phase de préparation prend beaucoup de temps et se termine par un message d'erreur (RequestEntityTooLarge ou End of Central Directory record could not be found). Si vous essayez d'importer l'archive compressée directement dans WSC, le message est le même.

Dans ce cas, l'archive doit être réduite:

  1. A l'aide de 7-Zip®, décompressez l'archive du pilote.

  2. Parmi les dossiers décompressés, sélectionnez le dossier marqué 64 (bits) (ou 32 bits si vos postes utilisateurs requièrent cette version).

  3. Recompressez ce dossier unique, qui devient le package du pilote.

  4. Ouvrez DriverPackageExtractor, en spécifiant le dossier 64 bits compressé dans le champ Driver archive.

  5. Cliquez sur Réparer le package et analyser.

Package du pilote non conforme

Si vous tentez d'ajouter un package de pilote non-conforme aux attentes de WSC, un message d'erreur s'affiche dans l'interface Ajouter un pilote d'impression : No certified .inf file found.

.
Dans ce cas, recommencez la procédure en veillant à ce que le pilote provienne bien du site officiel du fabricant et en utilisant DriverPackageExtractor.
