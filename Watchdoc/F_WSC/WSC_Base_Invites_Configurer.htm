WSC - Base Invités (Guests) - Configurer la base des invités

Principe

Dans Watchdoc, la gestion des Invités permet d'administrer une base dans laquelle sont enregistrés les Invités, utilisateurs non-inscrits dans l'annuaire de Watchdoc. Ainsi, ces utilisateurs de passage peuvent imprimer de manière sécurisée sans inscription préalable dans l'annuaire de Watchdoc.

Accéder à l'interface de configuration

  1. Depuis le Menu principal de WSC, cliquez sur l'entrée de menu Configuration :

  1. Dans l'interface Configuration avancée, cliquez sur le bouton Base des invités :




    è Vous accédez à l'interface Configuration de la base des invités.

Configurer la base des invités

Section Informations de connexion

  1. Dans cette section, indiquez les informations permettant d'établir la connexion avec la base dans laquelle sont enregistrés les comptes invités :

    • Type : sélectionnez dans la liste le type de base de données utilisée pour enregistrer les comptes invités ;

    • Serveur SQL : indiquez dans ce champ le serveur sur lequel se trouve la base de données utilisée pour enregistrer les comptes invités. Si vous ne connaissez pas le nom du serveur, cliquez sur le bouton pour parcourir la liste des serveurs existants et y sélectionner le serveur utilisé.

    • Authentification : sélectionnez le type d'authentification requise pour accéder au serveur SQL (SQL serveur ou Windows SSPI)

    • Nom d'utilisateur : si vous avez opté pour l'authentification SQL, saisissez dans ce champ le nom d'utilisateur autorisé à accéder à la base de données utilisée ; si vous avez opté pour l'authentification SSPI, cette information sera communiquée automatiquemement.

    • Mot de passe : si vous avez opté pour l'authentification SQL, saisissez dans ce champ le mot de passe associé à l'utilisateur autorisé à accéder à la base de données ; si vous avez opté pour l'authentification SSPI, cette information sera communiquée automatiquemement.

    • Base de données : dans le champ, saisissez le nom de la base de données utilisée pour enregistrer les comptes invités.

  2. Une fois les champs complétés, cliquez sur le bouton Tester la connexion pour vérifier que les informations fournies permettent d'accéder à la base de données utilisée pour enregistrer les comptes invités.

Section Code PUK généré automatiquement

L'invité peut disposer d'un code PUK Puk = Print User Key. Dans Watchdoc, il s'agit d'un code associé à un compte utilisateur pour permettre à ce dernier de s'authentifier dans un WES. Le code PUK est généré par un algorithme. L'utilisateur peut le consulter dans la page "Mon compte" de Watchdoc. Par souci de sécurité, nous déconseillons l'utilisation du code PUK et recommandons l'utilisation d'un login (compte utilisateur)/code PIN. ou d'un code PIN Le code PIN (Personal Identification Number) est un code comportant au moins 4 chiffres. Il est, par exemple, utilisé sur un téléphone mobile ou un smartphone équipés d'une carte SIM. Dans Watchhdoc, c'est un code qui, associé au nom d'utilisateur, constitue un moyen d'authentification sur un WES. Ce code peut être saisi et consulté depuis la page Mon Compte. Il est enregistré soit dans une table SQL, soit dans un fichier Json et peut être stocké tel quel ou sécurisé par hachage. pour s'authentifier dans Watchdoc. Ces codes sont saisis dans l'interface de création du compte invité. Si ces codes ne sont pas saisis manuellement, il sont générés automatiquement lors de la validation du compte invité. 
Lors de la génération, WSC ajoute le chiffre 9 comme préfixe du code PUK afin de discriminer les codes PUK de la base Invités d'autres codes PUK existants.

 

Dans cette section, vous indiquez le niveau d'algorithme à appliquer lors de la génération automatique du code PUK (quand ce code PUK est saisi manuellement lors de la création du compte invité, sa longueur n'est pas contrôlée) :

  • soit un algorithme de 7 chiffres (sécurité faible) ;

  • soit un algorithme de 11 chiffres (sécurité moyenne).
