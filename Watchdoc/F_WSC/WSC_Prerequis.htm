WSC Console de supervision - Prérequis

Prérequis système

Dimensionnement

Pour un fonctionnement optimal de la Console de Supervision il est recommandé de disposer au minimum des prérequis suivants :

  • Processeur : 1 cœur minimum

  • Mémoire vive : 512 Mo de disponibles

  • Disque dur : 256 Mo (+/- 1,5 Ko par files d’impression)

  • Port : Watchdoc Supervision Console utilise le port sécurisé 5756 en TCP.

Si la configuration comprend un pare-feu (firewall), ce dernier doit autoriser les accès sur le serveur hébergeant Watchdoc et Watchdoc Supervision Console via ces ports.

Droits spécifiques au compte gMSA

Dans un contexte exploitant un compte gMSA, le téléchargement de packages nécessaires à la gestion des pilotes et files d'impression requiert les droits suivants :

  • OpenPrinter: Aucun

  • GetPrinter: Aucun (lecture)

  • EnumPrinters: Aucun

  • ClosePrinter: Aucun

  • SetPrinter: Manage Printers

  • DeletePrinter: Manage Printers

  • AddPrinter: Manage Server

  • AddPrinterDriver: Manage Server + SeLoadDriverPrivilege

  • GetPrinterDriver: Manage Server

  • DeletePrinterDriver: Manage Server

  • UploadPrinterDriverPackage: Manage Server + SeLoadDriverPrivilege (souvent attribué à un administrateur)

  • InstallPrinterDriverFromPackage: Manage Server + SeLoadDriverPrivilege

  • AddMonitor: Manage Server + SeLoadDriverPrivilege

Prérequis organisationnel

Pour configurer une file d'impression dans WSC, il est obligatoire d'avoir préalablement configuré un emplacement auquel il est possible d'assigner des imprimantes physiques.

 

Si vous ne souhaitez pas ajouter d'emplacements spécifiques à la liste des emplacements par défaut, configurez au minimum l'emplacement Monde de sorte que des imprimantes physiques puissent être assignées à cet emplacement (cf. Configurer des emplacements - Paramètre Appareils). 

Prérequis logiciels

Framework

  • Watchdoc exploite la technologie .net Framework 4.8 de Microsoft (à télécharger ici). Ce framework, librement téléchargeable depuis le site de Microsoft® doit impérativement être installé.

Base de données

Les données des badges utilisateurs sont stockées dans un système de gestion de base de données relationnelle. Cette base doit être accessible depuis le serveur et le compte.

Les serveurs de gestion de bases de données relationnelles supportés sont :

  • Microsoft SQL Server (Configuré en mode mixte – SQL and Windows Authentication)

  • PostgreSQL

  • SQLite

Watchdoc peut se connecter à la base Microsoft® SQL Server uniquement avec un compte d’utilisateur SQL et non avec un compte d’utilisateur Windows®. 
Par ailleurs, si serveur SQL est distant, vérifiez que TCP/IP est activé dans SQL Server Configuration Manager.
En cas d'indisponibilité de la base de données, WSC peut fonctionner en mode dégradé. Dans ce cas, un message informe les administrateurs/utilisateurs de la perte de fonctionnalités qui ne sont plus complètement opérationnelles.

Annuaire LDAP

Les informations relatives aux utilisateurs stockées dans la base de statistiques sont obtenues depuis l’annuaire LDAP de l’entreprise.

Cet annuaire doit donc être accessible depuis le serveur hébergeant WSC.

Les annuaires supportés sont :

  • Active Directory®

  • Open LDAP®

Package d'installation

Le package d'installation de la Console de Supervision Watchdoc Supervision Console est téléchargeable depuis doc.doxense.

L’archive d’installation contient les fichiers d’installation du framework .NET 4.7.2 ainsi que le fichier Supervision Bootstrapper.exe.
