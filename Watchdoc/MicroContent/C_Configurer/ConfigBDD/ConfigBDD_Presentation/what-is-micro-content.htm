Watchdoc - Base de données WatchdocStats - SQL / PostgreSQL / SQLite

Compatibilités

Watchdoc est compatible avec :

  • MS SQL Server® (Express/Standard/Enterprise) 2012, 2014, 2016, 2017, 2019 et 2022 (cf. Prérequis du serveur de données) ;

  • PostgreSQL® 17
    Notez que Doxense® n'assure pas le support pour le paramétrage du SGBD PostgreSQL®.

  • SQLite® Watchdoc est compatible avec la version SQLite® distribuée dans le package d'installation de Watchdoc.
    Notez que SQLite fonctionnant comme une base de données locale, elle n'est recommandée que pour de petites configurations ou sur les serveurs slaves d'un domaine qui envoient leurs données vers la base de données SQL du serveur master.

 

Principe

  • Dans Watchdoc, l'activité d'impression est enregistrée dans une base de données statistique SQL® ou PostgreSQL® appelée Watchdocstats.

  • la fonction d'impression à la demande interserveur repose également sur une base de données statistique de type SQL ;

  • Cette base est alimentée grâce aux données collectées par Watchdoc via le protocole SNMP.

  • Elle peut-être utilisée à des fins d'analyse, mais aussi pour gérer les quotas d'impression.

  • Les données sont affichées dans l'interface d'administration de Watchdoc mais peuvent aussi être exploitées et exportées par WDIS.

  • Les données sont aussi exploitables par les utilisateurs dans leur page "Mon compte'.

  • La base de données de type SQL peut aussi être utilisée en guise d'annuaire :

    • Annuaire utilisateurs : dans de rares cas où l'organisation ne dispose pas d'un annuaire LDAP, elle peut utiliser une table SQL en guise d'annuaire utilisateurs (cf. Créer et configurer une base de données Utilisateurs).

    • Annuaire invités : WSC offre par défaut une base Invités (Guest) qui repose sur une table SQL. Cet annuaire permet de référencer les utilisateurs non présents dans l'annuaire de l'organisation mais qui doivent être reconnus par Watchdoc pour imprimer (par exemple, un partenaire de passage, un stagiaire, un prestataire missionné pour une durée déterminée, etc. (cf. Gérer la base Invités).

    • Codes d'impression (Code PUK) : cet annuaire permet de générer des codes d'impression PUK à l'attention des utilisateurs inscrits dans l'annuaire Entra ID (Microsoft Azure). (cf. Créer et configurer une base de codes PUK)

       

Il est donc nécessaire d'installer une base de données préalablement à l'installation de Watchdoc. Pour cela, vous pouvez consulter :

  • les Prérequis des serveurs de données

  • les Prérequis d'installation et packages MS SQL

  • Créer la base MS SQL

  • Configurer la base MS SQL

  • Créer la base Postgre SQL

 

Par ailleurs, comme les données de la base varient, il arrive que le traitement des requêtes soit ralenti en raison d'une fragmentation des index. Pour éviter cet inconvénient, nous vous recommandons de nettoyer régulièrement votre base en suivant le plan de maintenance fourni.

 

Il est parfois utile de restaurer la base de données.
