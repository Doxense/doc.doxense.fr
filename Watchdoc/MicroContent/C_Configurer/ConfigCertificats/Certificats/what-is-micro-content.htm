Point d'entrée du serveur web et certificats

Watchdoc échange des informations avec différents périphériques (imprimantes, MFP, lecteurs de cartes) grâce à des interfaces telles que WSC, les WES et autres API (dont la Print API pour WPC et Skyprint).

Pour sécuriser ces pages accessibles via Internet, Watchdoc utilise le protocole TLS/SSL (via les ports 5744, 5753 et 5754) reposant sur des certificats auto-signés.

 

Avant, c'est à l'aide de l'outil en ligne de commande WCM (Wachdoc Certificat Manager) que l'on pouvait gérer ces certificats ou d'autres, signés par une autorité de certification. WCM permettait aussi de diagnostiquer les anomalies impliquant les certificats et les protocoles SSL/TLS (cf. Signer un certificat avec WCM).

 

Depuis Watchdoc v. 6.1.0.5011, Watchdoc dispose d'une interface dédiée, plus simple à utiliser que l'outil WCM (qui reste néanmoins opérationnel). Pour accéder à cette interface, depuis le Menu principal, cliquez sur Configuration avancée > Serveur web.

 

Dans cette interface, vous pouvez modifier les points d'entrée du serveur web existants et en créer de nouveau. Attention, toutefois à ne pas désactiver le point d'entrée _crystal, sous peine de rendre Watchdoc inopérant. (cf. Configurer les points d'entrée du serveur web).

 

Vous pouvez aussi y gérer les certificats nécessaires à la sécurisation du serveur web de Watchdoc (cf. Gérer les certificats).
