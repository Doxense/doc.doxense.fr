Mode d'une file

Watchdoc exploite les files d'impression selon différents modes qu'il est possible de configurer (cf. Configurer une file).

File physique

Il s'agit d'une file associée à un périphérique d'impression (imprimante ou MFP).
Cette file est dédiée à un seul périphérique. 

File virtuelle

Il s'agit une file qui n'est pas attribuée à un périphérique précis, mais qui conserve le travail d'impression prêt à être envoyé à l'un des périphériques du réseau lorsque l'utilisateur en fait la demande.

 

Ce type de file est sollicité losque la fonction Impression à la demande Fonction qui permet à l'utilisateur de lancer ses travaux d'impression depuis son environnement de travail puis de débloquer ses impressions sur le périphérique de son choix. La validation de ses impressions peut être effectuée soit depuis l'interface web de Watchdoc, soit depuis le WES, après authentification par l'utilisateur. est activée, fonction qui permet à un utilisateur d'envoyer ses travaux d'impression sans désigner d'imprimante spécifique, puis de débloquer physiquement son impression sur l'imprimante de son choix après s'y être authentifié.

 

Comme la file virtuelle envoie des travaux d'impression à divers périphériques aux capacités différentes, nous vous recommandons de configurer cette file avec des capacités maximales (voir la section Périphériques : impression couleur, impression recto verso et grand format) afin d'optimiser son efficacité.
Pour plus de détails, cf. Configurer une file virtuelle.

File universelle

Il s'agit d'une file logique créée exclusivement pour la fonction Watchdoc Print Client (WPC). 
Elle permet aux utilisateurs de lancer leurs demandes d'impressions depuis WPC for Windows, WPC for Android ou Skyprint, puis de débloquer physiquement leurs impressions sur l'un des périphériques compatibles.

La file universelle utilise le spouleur Doxense (custom spooler) et n'utilise pas le spouleur MS Windows. Par ailleurs, elle n'existe pas sur le serveur MS Windows.

La file universelle requiert un pilote d'impression de type 3.

Pour plus de détails, cf. Configurer une file universelle.

File personnelle

Il s'agit d'une file associée à des périphériques d'impression connectés aux postes de travail via un port USB (ou port LPT1) et non en réseau. 

Les données d'impression de ce type de file peuvent être envoyées à Watchdoc à l'aide de WPC for Windows (qui a remplacé l'Agent local depuis Watchdoc v 5.4.1), préalablement installé sur le poste.

File proxy (proxy queue)

Lorsque Watchdoc est installé en domaine (configuration master/slaves) et que la fonction Impression interserveur est activée, une file proxy est une file gérée sur un serveur slave mise à disposition d'autres serveurs slaves.
La file proxy est alors visible dans la liste des files d'un serveur, bien qu'elle ne soit pas gérée par ce serveur. Pour voir les files proxy, il faut activer le filtre "proxy" dans la liste des files.
