Sécurisation des flux - Windows Data Protection Application Programming Interface (DPAPI)

1. Le spooler MS Windows est-il bien chiffré tout comme le spooler Doxense ?

Non, le spooler MS Windows, développé par Microsoft, ne permet le chiffrement des spools.

En conséquence, le spool n'est pas chiffré avant d'être pris en charge par le spooler Doxense..

Toutefois le flux SMB SMB (Server Message Block) est un protocole de communication utilisé principalement pour le partage de fichiers et d'imprimantes sur un réseau. Il permet aux applications d'accéder à des fichiers et à des ressources sur des ordinateurs distants. Les versions récentes, SMB v2 et SMB v3, offrent des fonctionnalités de chiffrement pour sécuriser les échanges entre les postes de travail et les serveurs, renforçant ainsi la protection des données. entre le poste de travail et le serveur peut être chiffré (cf. Configurer le client SMB pour exiger le chiffrement dans Windows).

 

2. Comment sont stockées les clés de chiffrement utilisées par Watchdoc dans la fonction de chiffrement des fichiers ?

Les clés sont des dérivés des mots de passe des comptes de services utilisés pour exécuter Watchdoc.
Ces clés sont générées aléatoirement lors de l'exécution de Watchdoc (sont-elles stockées ensuite ou ne servent-elles qu'1 fois ?)

 

3. Comment est sécurisé un travail d’impression envoyé vers l’imprimante avec Watchdoc ?

Le travail d'impression (transformé en spool d'impression) est chiffré par le spooler Doxense, puis déchiffré avant d'être envoyés à l'imprimante au moment d'être pris en charge par l'imprimante. De plus, entre le serveur et l'imprimante, la connexion est chiffrée à l'aide du certificat de l'imprimante.

 

4. Le protocole IPP est-il la seule option pour sécuriser le flux ?

Oui c'est la seule option, les constructeurs d'imprimantes ne permettent pas d'autres solutions pour les éditeurs tiers.

 

5. Est-il possible d’activer le chiffrement entre Watchdoc et la base de données SQL ?

Les données les plus sensibles présentes dans la bdd SQL sont les comptes utilisateurs et les titres des documents.

Watchdoc utilise les librairies .net framework pour se connecter à la bdd SQL. Si le port SQL est sécurisé par un certificat, la connexion est également sécurisée.

 

Dans le cadre du RGPD nous disposons d'un outil pour anonymiser les informations Watchdoc (cf. Anonymisation des travaux d'impression avec WatchdocGDPRComplier).

Attention, si vous anonymisez ces données, elles ne seront plus utilisables dans les rapports statistiques (filtre par utilisateur, classement des utilisateurs les plus consommateurs, etc).
