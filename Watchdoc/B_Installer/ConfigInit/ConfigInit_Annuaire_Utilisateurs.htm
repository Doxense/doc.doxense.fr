Watchdoc - Configurer l'annuaire utilisateurs

Watchdoc nécessite l'installation d'un annuaire. Cet annuaire peut avoir été installé avant la configuration initiale de Watchdoc. Dans ce cas, passez outre le paramétrage en cliquant sur Suivant.

Section Configuration et Droits

Watchdoc exploite un annuaire dans lequel sont enregistrés les usagers. Cet annuaire est propre à l'architecture réseau de l'organisation.

  • Domaine Windows : par défaut, si le serveur est déjà membre du domaine, Watchdoc détecte automatiquement les informations du domaine et les affiche dans ce champ. Si aucune information n'y apparaît, saisissez dans ce champ le nom du domaine Windows auquel appartient le serveur Watchdoc.
    N.B. : dans le cas d'un annuaire Active Directory principal, le nom du domaine Windows doit être identique au nom du Domaine NT (DN).

  • Type de domaine : sélectionnez dans la liste le type de domaine de l'annuaire. 

Le type Windows NT (SAM) ne peut être utilisé que lorsque le contrôleur de domaine est un MS Windows N.T. 4.0 ou lorsque le serveur ne fait partie d'aucun domaine.
Le type MS Active Directory (LDAP) est utilisé dans le cas où le domaine est contrôlé par un serveur MS Windows 2000 ou supérieur.
Il est également possible d'utiliser un serveur Novell, OpenLDAP ou générique LDAP v3.

  • Description : par défaut, si le serveur est déjà membre du domaine, Watchdoc détecte automatiquement les informations saisies en description et les affiche dans ce champ. Au besoin, vous pouvez modifier cette description.
    Si vous disposez d'un contrôleur de domaine secondaire, il est conseillé de ne pas indiquer de serveur LDAP précis. Ainsi, en cas de panne d'un annuaire, le contrôleur de domaine peut en utiliser un autre.

  • Port : dans le cas où vous avez rempli le champ Serveur, indiquez dans ce champ le port d'accès à ce serveur. Par défaut, il s'agit du port 389.

 

Si le domaine est de type LDAP, le champ suivant doit être complété :

  • DN : indiquez dans ce champ le Default Naming Context du serveur.

    Si le compte utilisé n'est pas situé sous "CN=Users,", il est impératif de préciser le DN complet (ex: "CN=compte_readonly,OU=test,DC=...,DC=...").

  • Utiliser le compte de connexion suivant : cochez la case si l'annuaire nécessite une authentification. Dans ce cas, indiquez le compte d'accès à l'aide du compte (identifiant / mot de passe) saisi dans les champs suivants :

  • Identifiant : nom de l'utilisateur habilité à accéder à l'annuaire ;

  • Mot de passe : mot de passe attribué à l'utilisateur habilité à accéder à l'annuaire.

Section Cache

  • Gérer un cache : cochez la case pour activer l'enregistrement des requêtes dans un ficher cache du serveur. Cette fonction peut être utile pour éviter à Watchdoc de solliciter l'annuaire et améliorer ainsi ses performances, mais aussi en cas d'indisponibilité ponctuelle de l'annuaire.

  • cliquez sur le bouton pour déclarer l'annuaire.

Configurer la section Configuration automatique

  • WES : cochez la case pour permettre la création et la configuration a minima des profils WES de toutes les marques constructeurs qu'il est possible d'activer dans Watchdoc. Cette opération vous permettra de gagner du temps lors du paramétrage des WES.

  • Utilisateurs invités : cochez la case pour permettre la création de la base des invités (base Guests) dans la Console de Supervision de Watchdoc (WSC).

  • Emplacement : (v6.0.0.4731) cochez la case si vous ne souhaitez pas utiliser et gérer les emplacements (anciennement "Géosites" dans les versions antérieures à la V6). Dans Watchdoc, un emplacement est un libellé appartenant à une liste structurée de sites géographiques permettant de localiser physiquement des éléments de configuration du parc d'impression (périphériques d'impression, serveurs, files, etc.). La liste des emplacements est administrable depuis l'interface d'administration de Watchdoc. Les emplacements sont nécessaires à la fonctionnalité Watchdoc Print Client.. Si la case est cochée, l'emplacement générique Monde est sélectionné par défaut dans Watchdoc et WSC.
