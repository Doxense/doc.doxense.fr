Watchdoc - Service - Configurer les bases de données

Principe

Dans cette interface, vous indiquez les informations relatives à la (aux) base(s) de données nécessaire(s) à Watchdoc et les connecter :

  • base de données statistiques (watchdocstats), obligatoire ;

  • base de données dédiée à la fonction Quotas (watchdocpmv), facultative.

Cette configuration suppose qu'une base de données dédiée ait été créée préalablement (cf. Créer une base de données MS SQL® ou Créer une base de données PostgreSQL®).

Configurer la base Statistiques

  • Sous-section Connexion

    • Type : dans la liste, sélectionnez le type de base de données installée dans votre infrastructure réseau pour collecter les statistiques relatives à l'utilisation du service Watchdoc.

      Watchdoc supporte les systèmes suivants :

      • Microsoft SQL Server® 2012 jusquà v. 2022

      • Microsoft SQL Desktop Engine,

      • PostgreSQL 17

      • SQLite 3.x

    • Serveur SQL : dans la zone, indiquez le chemin de localisation de la base de données. Si vous ne connaissez pas la localisation de la base de données, cliquez sur le bouton . Cette fonction parcourt le réseau et renvoie la liste des serveurs SQL qui y sont installés. Sélectionnez dans cette liste le serveur SQL désigné pour les statistiques Watchdoc.

    • Nom d'utilisateur : indiquez dans cette zone le nom de l'utilisateur habilité à gérer la base de données. Pour plus de sécurité, il est recommandé de créer un compte propre à la base des statistiques ;

    • Mot de passe : indiquez dans cette zone le mot de passe associé au nom de l'utilisateur ;

    • Base de données : indiquez dans cette zone le nom de la base de données utilisée. Par défaut, le nom est Watchdocstats.

    • Chemin : ce paramètre n'est activé que dans le cas d'un annuaire de type SQLite. L' information affichée est le chemin du dossier (data) dans lequel est créée la base de données SQLite par défaut.


  • Sous-section Opérations :

    • Bouton Créer une nouvelle base de données : cliquez sur ce bouton dans le cas d'une première installation de Watchdoc afin de créer la base de données statistique ;

Configurer la base Quota (Watchdocpmv)

La section Quotas regroupe les informations relatives à la base de données collectant les informations des quotas.

  • Sous section Activation

    • Activer la gestion des quotas : cochez cette case si vous souhaitez utiliser les quotas pour le paiement des impressions.

  • Sous-section Connexion

    • Type : dans la liste, sélectionnez le type de base de données installée dans votre infrastructure réseau pour collecter les données relatives à l'utilisation du PMV
      Watchdoc supporte les systèmes suivants :

      • Microsoft SQL Server® 2012 jusqu'à v. 2022

      • Microsoft SQL Desktop Engine,

      • PostgreSQL 17,

      • SQLite (version embarquée dans le package d'installation Watchdoc).

    • Serveur SQL : saisissez le chemin de localisation de la base de données ou cliquez sur le bouton pour parcourir la liste des serveurs SQL installés sur le réseau. Puis sélectionnez serveur SQL désigné pour les statistiques Watchdoc.

    • Nom d'utilisateur : saisissez le nom de l'utilisateur habilité à gérer la base de données. Pour plus de sécurité, il est recommandé de créer un compte propre à la base des Quotas ;

    • Mot de passe : saisissez le mot de passe de l'utilisateur habilité ;

    • Base de données : saisissez le nom de la base de données utilisée ou cliquez sur le bouton pour parcourir la liste des bases déclarées dans le serveur. Par défaut, la base servant à gérer les quotas s'appelle Watchdocpmv ;

    • Chemin : ce paramètre concerne exclusivement une base de type SQLite. L'information affichée est le chemin du dossier (data) dans lequel est créée la base de données SQLite par défaut.



  • Sous-section Opérations : dans cette section, cliquez sur l'un des deux boutons :

    • Créer une nouvelle base de données dans le cas d'une première installation de Watchdoc afin de créer la base de données des Quotas. Complétez ensuite l'interface Création d'une base de données :

      • Options : cochez cette case si vous ne souhaitez pas créer une base complète, mais pour créer les tables nécessaires à la gestion des Quotas dans une base existante ;

      • Identification : si l'accès à la base de données nécessite une authentification autre que le compte par défaut WatchdocUser, cochez la case Utiliser un autre compte, puis saisissez l'identifiant et le mot de passe du compte habilité à accéder à la base ;

      • Chemin : ce paramètre concerne exclusivement une base de type SQLite. L'information affichée est le chemin du dossier (data) dans lequel est créée la base de données SQLite par défaut.
