Créer des règles et des filtres - Procédure

Etapes

Les étapes d'application d'une règle sont les suivantes :

  1. concevoir les rôles (s'ils entrent dans la composition de la règle) ;

  2. concevoir un ou plusieurs filtres ;

  3. déclarer la file d'impression ;

  4. concevoir et appliquer la règle sur la file d'impression.

     

Une règle peut être appliquée sur un groupe de filesou sur une file en particulier. Le paramétrage d'une règle sur une file ou un groupe de files s'effectue dans la même interface.

Créer un filtre

  1. Depuis le Menu principal de l'interface d'administration de Watchdoc®, dans la section Configuration, cliquez sur Filtres.

→ Vous accédez à la liste des filtres déclarés et filtres prédéfinis par défaut dans Watchdoc®.

 

  1. Pour créer un filtre, dans la liste des filtres, cliquez sur le bouton Créer un nouveau filtre :



→ vous accédez à l'interface Création d'un filtre constituée de plusieurs sections.

Configurer le Filtre

Configurer la section Filtre

Indiquez dans cette section les informations générales concernant le filtre :

  • Nom : attribuez un nom explicite au filtre. Ce nom est celui qui s'affiche dans l'interface d'administration .

  • Message : saisissez un message expliquant les règles appliquées dans le filtre. Ce message est destiné aux utilisateurs pour justifier, par mail ou par notification, les préférences d'impression imposées, la redirection ou le rejet de son impression. Par exemple : "Vous ne disposez pas du droit d'imprimer les mails en couleur" ; "En tant que public extérieur, vous ne disposez pas du droit d'imprimer en couleur", etc.

Configurer les conditions

Le filtre repose sur une ou plusieurs conditions qui doivent être vérifiées pour que les travaux d'impression puissent être discrimés.

Par exemple : "Le document est en couleur" ; "Le jour de la semaine est..." ; "L'utilisateur est identifié", etc. sont des conditions prédéfinies dans Watchdoc®.



  • Condition : sélectionnez dans la liste une première condition entrant dans la composition du filtre. Les conditions portent sur :

    • des caractéristique relatives aux documents : couleur ou monochrome, nombre de pages, impression ou photocopie, format de papier, etc.

    • des caractéristique relatives aux utilisateurs : répondant à un nom particulier, appartenance à un groupe, à un domaine, à un rôle, etc.

    • des caractéristique relatives à la date et/ou à l'heure : jour spécifique, horaires d'ouverture, avant ou après une heure précise, etc. (la valeur "Pendant les horaires d'ouverture" correspond aux horaires indiqués dans la section   de la Configuration générale ;

    • le poste de travail : possède une IP précise, répond à un nom DNS spécifique, etc ;

    • le type de refacturation : le code de refacturation correspond à un client précis, le coût est supérieur ou inférieur à une certaine somme, etc. ;

    • l'état de la file d'impression : la file contient un tag spécifique, la file appartient à un groupe de files spécifique, etc. ;

    • un autre critère (divers) : le document doit valider ou ne pas valider un filtre déjà prédéfini.

  • Paramètre : lorsque le champ de saisie est actif, saisissez la valeur correspondant à la condition qui doit être vérifiée.

Certains paramètres doivent respecter des formats précis, cf. liste ci-dessous.

  • Si vrai : sélectionnez l'action effectuée lorsque la condition est vérifiée :

    • Continuer : la vérification continue et passe à la condition suivante ;

    • Sélectionner : l'élément analysé est sélectionné puisqu'il répond à la condition ;

    • Rejeter : la vérification s'arrête et l'élément analysé est rejeté puisqu'il ne répond pas à la condition.

       

    • Si faux : sélectionnez l'action effectuée lorsque la condition n'est pas vérifiée :

    • Continuer : la vérification continue et passe à la condition suivante ;

    • Sélectionner : l'élément analysé est sélectionné puisqu'il ne répond pas à la condition ;

    • Rejeter : l'élément analysé est rejeté puisqu'il ne répond pas à la condition et la vérification s'arrête.

    Pour une condition, vous ne devez pas avoir une action identique dans les colonnes Si-Vrai et Si-Faux d'une même ligne.
    Si une ligne ne contient aucune action Continuer, la vérification s'arrête et les conditions suivantes ne sont pas évaluées.

    Format des paramètres

    Lors de la conception des règles, certains paramètres doivent respecter des formats précis :

    Utilisateur

    Le nom de l'utilisateur est... DOMAINE/login

    vérifie le compte utilisateur dans l'annuaire nommé "Domaine".
      login vérifie le compte utilisateur dans l'annuaire par défaut.
    Le nom de l'utilisateur vérifie le masque... Etu* le compte commence par Etu
      DOMAINE\groupe vérifie le groupe dans l'annuaire nommé "Domaine".
      groupe vérifie le groupe dans l'annuaire par défaut.
    L'utilisateur est dans le domaine... DOMAINE Vérifie l'utilisateur dans un annuaire précis.

     

    Dates

    La date est... JJ  jour (du mois et de l'année courants) au format français
      JJ/MM jour et mois (de l'année courante) au format français
      JJ/MM/AAAA date précise au format français
      DD jour (du mois et de l'année courants) au format international
      MMDD jour et mois (de l'année courante) au format international
      AAAAMMJJ date précise au format international
      JJ/MM - JJ/MM date est comprise entre... et ... (bornes incluses)
    Le jour de la semaine est... (n'est pas...)

    1 à 7

    		 jours représentés par des chiffres de 1 = lundi à 7 = dimanche. Par exemple : 1 - 5 signifie "du lundi au vendredi.

     

    Type de document

    Le type du document est... e-mail email,
      pdf document PDF
      word document MS Word®
      excel document MS Excel®
      image tous types d'images
      web documents html.

     

    Heures

    L'heure est ... 6pm 18h, 6 heures de l'après-midi
      18 18h, 6 heures de l'après-midi
      18h 18h, 6 heures de l'après-midi
      18:00 18h, 6 heures de l'après-midi
    L'heure est comprise entre... et... 17-19 heure comprise entre 17h et 19h
      19h-20h30 heure comprise entre 19h et 20h30
      19:00 - 20:30 heure comprise entre 19h et 20h30
      17-19 heure comprise entre 17h et 19h

     

    DNS

    Le nom DNS est... poste1.domaine.fr Le nom DNS du poste de travail.
    Le nom DNS vérifie le masque... *-dev.ad-doxense.fr le nom DNS du poste de travail vérifie un masque : il se termine en "-dev" et se trouve dans le domaine DNS "ad-doxense.fr"

     

    Adresse IP

    L'adresse IP est... 192.168.1.10 Adresse IP précise du poste de travail
    L'adresse IP est dans la plage... 192.168.1.* Tous les postes dont l'adresse IP est comprise entre 192.168.1.0 et 192.168.1.255
