Watchdoc Agent SSO - Configurer

Configurer les files d'impression et/ou groupes de files

Pour activer l'Agent SSO sur une file :

  1. accédez à l'interface d'administration de Watchdoc en tant qu'administrateur ;

  2. depuis le Menu principal, section Exploitation, cliquez sur Files d'impression, emplacements, groupes de files & pools :

  3. dans la liste des files, sélectionnez celle pour laquelle vous configurez l'Agent SSO ;

  4. dans l'interface de configuration de la file, cliquez sur le bouton Editer les propriétés ;

  5. dans l'interface Propriétés de la file d'impression, section Mode Expert, paramètre Identité, cochez la case Etablit la correspondance avec les utilisateurs authentifiés par logiciel SSO

Installer Watchdoc Agent SSO

Configurer WatchdocSSOAgentDeployer.bat

Pour installer l'Agent Watchdoc SSO :

  1. dans un répertoire réseau accessible de tous les postes clients qui nécessitent l'Agent SSO, créez un dossier dédié à cet Agent SSO (par ex. : C:\temp\WatchdocSSO Agent) ;

  2. décompressez l’archive WatchdocSSOAgent.rar (fournie par Doxense) et enregistrez les fichiers décompressés dans ce dossier dédié ;

  3. ouvrez le fichier WatchdocSSOAgentDeployer.bat à l'aide d'un éditeur de texte ;

  4. dans ce fichier .bat , personnalisez les valeurs des set de sorte qu'elles correspondent aux valeurs de votre environnement :

    • set networkLocation=C:\Temp\WatchdocSSOAgent (chemin vers le dossier

    • ssoAgent sur le répertoire partagé réseau) ;

    • set watchdocServerName=localhost

    • set watchdocServerPort=5754

    • set watchdocServerSecure=false

    • set identityIgnoreDomain=true

 

Les postes clients qui nécessitent l'Agent SSO sont ensuite configurés pour exécuter le fichier .bat lors de leur démarrage.

 

Etablir la connexion entre le logiciel SSO et WatchdocAgentSSO

Pour établir la connexion entre le logiciel SSO et WatchdocAgentSSO, demandez à l'éditeur du logiciel SSO installé sur le poste client (comme SSOx, par exemple) de configurer son logiciel pour qu'il lance les deux lignes de commandes suivantes :

  • Logon SSO : “C:\temp\Watchdoc SSO Agent\WatchdocSSOAgent.exe” /standalone/event LogOn /user “DOMAIN\USER” (login de l’utilisateur connecté.avec nom de domaine facultatif ou obligatoire dans un environnement multi-domaine) ;

  • Logoff SSO : “C:\temp\Watchdoc SSO Agent\WatchdocSSOAgent.exe” /standalone/event LogOff

Dépannage

Lorsque l'Agent SSO fonctionne correctement, les données suivantes figurent dans le fichier de trace Watchdoc :

  • logon (authentification) ;

  • logoff (déconnexion) ;

  • minute event (au moins un événement une fois l'authentification effectuée) ;

  • événement lorsqu'un travail d'impression est associé à un utilsateur de l'A.D.

 

La base Watchdoc jsonDB s'enrichit alors d'une liste des événements survenus sur les postes : %WATCHDOC_PATH%\Data\SSO.jsdb\Hosts (pour chaque poste client, un fichier json contenant la liste des événements).
