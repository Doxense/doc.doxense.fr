Watchdoc - Supprimer plusieurs files d'impression contrôlées par Watchdoc

Contexte

Vous souhaitez supprimer plusieurs files d'impression contrôlées par Watchdoc.

Procédure

Il n'existe pas d'outil dédié à la suppression en masse de files d'impression dans Watchdoc. Pour supprimer en masse des files d'impression, procédez comme suit :

  1. effectuez une copie de sauvegarde du fichier \data\config.xml ;

  2. ouvrez un éditeur de texte (comme Notepad, SublimeTexte, par exemple) en tant qu'administrateur ;

  3. ouvrez le fichier config.xml en spécifiant l'encodage UTF-8 ;

  4. supprimez les blocs <queue>...</queue> correspondant aux files que vous souhaitez supprimer ;

  5. arrêtez le service Watchdoc ;

  6. enregistrez le fichier de configuration avec la modification apportée ;

  7. redémarrez le service Watchdoc ;

  8. dans la liste des files contrôlées, vérifiez que les files suppprimées n'apparaissent plus.

Notes

En fonction de l'objectif visé, il existe des choix lorsque vous supprimez une file d'impression :

  • S'il s'agit d'une file associée à un périphérique d'impression supprimé définitivement ou remplacé par un autre modèle, vous souhaitez l'externaliser. La configuration et les actifs de la file (numéro de série, modèle, fournisseur, fonctionnalités, etc.) seront toujours utilisés pour générer des statistiques. Sinon, ils sont remplacés par des données génériques qui pourraient ne pas correspondre aux caractéristiques de l'appareil.

  • If it's a queue that removed to be added again later (you're troubleshooting something or you changed the printer queue's name*), you want to remove it from the configuration. If you externalize it, when adding the queue again the "queue identifier", also called "queueID", will already be taken in the configuration file so Watchdoc will suffix it with a "1". (example : "MYPRINTSERVER.MYPRINTER1"). Consequence is that Watchdoc will consider this queue as new and won't inherit the statistics of the previous one.

  • S'il s'agit d'une file supprimée pour être ajoutée ultérieurement (vous dépannez un périphérique ou vous avez changé le nom de la file d'impression), vous souhaitez la supprimer de la configuration. Si vous l'externalisez, lors de l'ajout de la file d'attente, le "queueID" (identifiant de la file) figurerant déjà dans le fichier de configuration, Watchdoc le suffixera d'un "1". (exemple : "MYPRINTSERVER.MYPRINTER1"). En conséquence, Watchdoc considérera cette file comme nouvelle et n'héritera pas des statistiques de la précédente.

     

Il est tout à fait possible d'ajouter une file dans Watchdoc et de lui attribuer un identifiant (QueueID). Cependant, vous ne pouvez pas modifier l'identifiant d'une file déjà contrôlée par Watchdoc. Nous ne disposons pas, pour le moment, d'outils permettant de migrer les données d'une file vers une autre, mais il est relativement simple pour qui connaît les requêtes SQL de le faire manuellement.
