Watchdoc - Gérer les sites distants

Principe

En fonction des contraintes techniques existantes et de la facilité que l'on souhaite proposer à l'utilisateur, il est possible de gérer les sites distants de différentes manières avec Watchdoc.

 

Prenons l'exemple d'une entité installée en métropole dont les agences sont dispersées géographiquement dans les DROM.
Son architecture technique d'impression inclut :

  • un serveur maître (master) installé en métropole ;

  • d'autres serveurs (qui dépendent du maître) installés dans les agences situées en DROM. 

 

En matière de gestion de l'impression avec Watchdoc, plusieurs configurations sont possibles.

  1. Le mode normal repose sur la configuration de files d'impression prédéfinies et préconfigurées sur le serveur maître Watchdoc et mises à disposition de tous les utilisateurs via les autres serveurs (slaves) Watchdoc installés dans les DROM.

  2. Le mode mixte repose sur le mode normal associé à WPC for Windows. Ce dernier module permet de configurer une file universelle (tagguée “remote”) qui peut ensuite être installée en un clic sur tous les postes utilisateurs. Ce dernier dispose ainsi, en un clic, d'une file d'impression totalement configurée instantanément utilisable. 

  3. le mode distant repose sur le mode normal associé à WPC for Windows qui permet d’installer une file d'impression prédéfinie et préconfigurée tout en autorisant l'impression directe. Ainsi, en cas de latence importante de communication entre le serveur installé localement et le serveur maître, l'utilisateur peut opter pour l'impression directe. Dans ce cas, les utilisateurs impriment sans solliciter le serveur Watchdoc.

    Pour que les données statistiques du périphérique servant à l'impression directe puissent cependant être collectées à des fins statistiques, il est nécessaire de configurer dans Watchdoc la file qui servira à l'impression directe. Comme il s'agit d'un mode dégradé et que Watchdoc n'est pas sollicité, les données statistiques ne sont pas aussi détaillées que celles qui sont habituellement collectées par Watchdoc. Néanmoins, il est possible de collecter a minima des informations fournies par le spouleur MS Windows (printinfo2 : titre du document, nombre de pages, couleur (couleur ou monochrome) et utilisateur).

Procédures de configuration

Configuration en mode normal sans WPC for Windows

  1. Sur le serveur maître, créez vos files préconfigurées (cf. Configurer une file d'impression).

  2. Déployez Watchdoc sur les autres serveurs (slaves) pour y installer la configuration des files d'impression configurées sur le serveur maître. Les files seront ainsi synchronisées.

 

Configuration en mode mixte avec WPC for Windows

Ce mode permet à l'utilisateur d'installer sur son poste une file d'impression en cliquant sur le bouton "Installer" de WPC for Windows. Il récupère ainsi facilement sur son poste de travail la configuration de la file fournie par le serveur Watchdoc.

 

  1. Sur le serveur maître, créez vos emplacements et configurez-les pour WPC (cf. WPC for Windows - Configurer l'emplacement) en précisant les paramètres additionnels.

  2. Associez les serveurs (maître et autres) à leur emplacement (cf. Associer un serveur à un emplacement).

  3. Configurez au moins une file universelle destinée à WPC for Windows. Ainsi depuis WPC for Windows, l'utilisateur pourra installer la file en cliquant sur le bouton "Installer une file". Vous pouvez y ajouter le tag "remote" pour préciser son utilisation. 

  4. Configurez WPC for Windows sans autoriser l'impression directe (section Service > Impression directe).

  5. Installez WPC for Windows sur tous les postes de travail (cf. Installer WPC for Windows sur le poste utilisateur).

 

Configuration en mode indirect

Ce mode permet à l'utilisateur d'installer sur son poste une file d'impression en cliquant sur le bouton "Installer" de WPC for Windows. Il récupère ainsi facilement sur son poste de travail la configuration de la file fournie par le serveur Watchdoc.
Mais si les serveurs sont hors-service ou que les délais de communication entre les serveurs sont trop longs, l'utilisateur peut imprimer directement sur la file universelle. 

 

  1. Sur le serveur maître, créez vos emplacements et configurez-les pour WPC (cf. WPC for Windows - Configurer l'emplacement) en précisant les paramètres additionnels.

  2. Associez les serveurs (maître et autres) à leur emplacement (cf. Associer un serveur à un emplacement).

  3. Configurez au moins une file universelle destinée à WPC for Windows. Ainsi depuis WPC for Windows, l'utilisateur pourra installer la file en cliquant sur le bouton "Installer une file". Vous pouvez y ajouter le tag "remote tcpip" pour préciser son utilisation. 

  4. Configurez WPC for Windows en autorisant l'impression directe (section Service > Impression directe).

  5. Installez WPC for Windows sur tous les postes de travail (cf. Installer WPC for Windows sur le poste utilisateur).
