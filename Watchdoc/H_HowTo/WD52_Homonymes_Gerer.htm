Annuaires - Gérer l'homonymie

Préambule

Comme Watchdoc permet la gestion des impressions propres à chaque utilisateur d'une entité, le cas spécifique de comptes utilisateurs homonymes peut présenter une difficulté de configuration.

Watchdoc intègre une gestion de l'homonymie reposant sur la configuration des annuaires META et CARDS ainsi que sur le paramétrage des files d'impression.

Prérequis

Pour permettre la gestion des homonymes,

  • il est impératif que le pilote (driver) du périphérique soit capable d'inscrire le domaine de l'utilisateur dans le fichier de spool ;

  • il est nécessaire d'établir un audit de faisabilité tenant compte du contexte technique et organisationnel dans lequel la problématique d'homonymie s'inscrit (architecture réseau, marques et modèles des périphériques, nombre d'annuaires, politique d'impression, etc.).

Principe

Généralement, lors de la création des comptes dans un annuaire (domaine), l'administrateur réseau différencie les utilisateurs qui portent le même nom en ajoutant l'initiale de leur prénom dans le login (ou l'adresse mail).

Ainsi, Adrien Dupont a le login a.dupont et Emma Dupont a le login e.dupont. Et si les initiales des prénoms sont identiques, l'administrateur trouve un élément permettant de discriminer les comptes (par exemple, Alexandre Dupont, arrivé après Adrien, se verra attribuer le compte al.dupont ou a.dupont2).

 

Chaque utilisateur peut ainsi imprimer ses documents sans ambiguïté. 

 

Or, il arrive que l'homonymie n'ait pas été anticipée (rare) ou, plus fréquemment, qu'il soit nécessaire de fusionner en un seul annuaire plusieurs annuaires distincts. C'est le cas lors du rapprochement de plusieurs entreprises qui mutualisent leurs ressources informatiques.

 

Dans ce cas de figure, il risque d'y avoir confusion en termes de gestion des impressions si aucune modification n'est possible au niveau des comptes utilisateurs. 

En effet, lors du traitement des impressions par Watchdoc, les comptes utilisateurs étant tous versés dans l'annuaire META, deux comptes homonymes ne peuvent plus être distingués l'un de l'autre car leur domaine (annuaire) initial est remplacé par le nom de domaine META.

 

Ne pas tenir compte de l'homonymie exposerait les utilisateurs au risque de voir leurs demandes d'impressions débloquées par leur homonyme ce qui, pour certains documents sensibles, poserait problème.

 

A partir de la v5.2, Watchdoc intègre une fonction permettant de définir les différentes manières de gérer les domaines afin de tenir compte des homonymes.

L'homonymie évoquée ici n'est pas une homonymie pure portant sur des utilisateurs ayant les mêmes prénoms et nom, mais sur l'homonymie des comptes utilisateurs plus précisément (Adrien ou Alice Dupont, qui ne sont pas des homonymes, possèdent tous deux un compte a.dupont).

Procédure

Pour activer la gestion de l'homonymie dans Watchdoc, il est nécessaire d'activer la gestion dans l'annuaire META (et dans CARDS si vous utilisez cet annuaire), puis de configurer les files ou groupes de files pour préciser le mode de gestion des homonymes.

 

  1. depuis le Menu principal de Watchdoc, section Exploitation, cliquez sur Annuaires utilisateurs ;

  2. dans la liste des annuaires, cliquez sur le bouton de l'annuaire META pour en éditer les propriétés :

     

  1. dans l'interface Configuration de l'annuaire, section Sous-annuaires, cochez la case Conserver le domaine initial des utilisateurs (afin de ne pas remplacer le domaine initial par le domaine META, ce qui risquerait de créer des homonymes impossibles à discriminer (EntA\a.dupont et EntB\a.dupont ne deviennent pas tous deux META\a.dupont) ;

  2. cliquez sur le bouton pour valider la configuration de l'annuaire META ;

  3. revenez au Menu principal de Watchdoc®, puis, dans la section Exploitation, cliquez sur Files d'impression ;

  4. dans la liste Files d'impression, sélectionnez soit un groupe de files, soit une file en particulier. Si vous choisissez un groupe de files, la configuration s'appliquera à l'ensemble des files incluses dans le groupe, mais vous pourrez modifier le paramétrage de chaque file ultérieurement.

  5. cliquez sur le bouton Editer les propriétés ;

  1. dans l'interface Propriétés du groupe de files ou Propriétés de la file, section Mode Expert, configurez le paramètre Domaine en sélectionnant le bouton correspondant à votre choix :

 

  • Utiliser le même paramètre que le groupe (paramètre uniquement disponible pour une file) : si la file appartient à un groupe, cochez cette case pour que la file hérite des paramètres du groupe et se comporte donc de la même manière pour gérer les homonymes ;

  • Ignorer le domaine du spool et toujours utiliser le domaine par défaut : cochez cette case pour ne pas tenir compte du domaine initial. Les comptes enregistrés dans l'annuaire META dépendent tous du domaine META. Cette option ne permet donc pas de distinguer les homonymes ;

  • utiliser le domaine du spool si présent, utiliser le domaine par défaut sinon : cochez cette case pour tenir compte du domaine initial en priorité ou du domaine par défaut si le domaine initial n'existe pas. Les comptes enregistrés dans l'annuaire META gardent ou non l'information de leur domaine initial. Cette option expose au risque de faire débloquer un travail d'impression par un utilisateur homonyme à l'utilisateur qui a lancé l'impression ;

  • utiliser le domaine du spool si présent, rejeter le travail sinon : cochez cette case pour conserver en toute circonstance l'information du domaine initial. Dans le cas où l'information relative au domaine initial est absente (car certains pilotes ne permettent pas de conserver cette information), Watchdoc® considère qu'il ne peut discriminer le propriétaire et n'imprime donc pas le travail. C'est l'option qui offre la garantie maximale de distinguer les homonymes, mais qui présente l'inconvénient de faire échouer une impression. De plus, l'information relative à l'échec de l'impression ne peut être communiquée au demandeur, puisque ce dernier ne peut être distingué de son (ou ses) homonyme ;

  • toujours utiliser le domaine suivant : cochez ce bouton et sélectionnez dans la liste déroulante un domaine spécifique à utiliser en priorité. Cette option permet de distinguer les utilisateurs de différents domaines en imposant un domaine.

 

  1. Cliquez sur le bouton pour valider le paramétrage du groupe de file ou de la file.

  2. Testez l'impression à partir de comptes homonymes pour vérifier l'efficacité du paramétrage.
