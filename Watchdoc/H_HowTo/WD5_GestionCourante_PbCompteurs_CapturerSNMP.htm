Watchdoc - SNMP - Capturer des données SNMP

Principe

Dans Watchdoc, les informations SNMP Simple Network Management Protocol (Protocole Simple de Gestion de Réseau) est un protocole de communication qui permet aux administrateurs réseau de gérer les équipements du réseau, de superviser et de diagnostiquer des problèmes réseaux et matériels à distance. (Source : Wikipedia) collectées sur le périphérique d’impression sont affichées dans l’interface d'administration de la file d'impression. 

 

Pour accéder aux données SNMP :

  1. depuis le Menu principal, section Exploitation, cliquez sur Files d'impression, groupes de files & pools ;

  2. dans la liste des files, cliquez sur la file que vous souhaitez analyser ;

  3. dans l'interface de gestion de la file, sous l'onglet Etat sont affichées les informations relatives à la file, dont les données des compteurs (encart du haut, à droite) ;

Questions préalables à l'analyse des données SNMP

Afin d'analyser les informations SNMP affichées, il convient de vérifier les paramètres suivants :

  1. pour vérifier que le contrôle SNMP est activé et configuré sur la bonne adresse I.P. rendez-vous dans l'interface d'administration de Watchdoc ;

  2. depuis le Menu principal, section Exploitation, cliquez sur Files d'impression, groupes de files & Pools ;

  3. dans la liste des files, cliquez sur la file que vous souhaitez analyser ;

  4. depuis l'interface de gestion de la file, cliquez sur le bouton Editer les propriétés :

  5. la section Monitoring, cochez la case correspondant au paramètre SNMP : activer le monitoring... ;

  6. dans le champ Adresse IP, vérifiez la validité de l'adresse IP saisie :

Dans le cas d'une configuration en DHCP, l'adresse IP du périphérique peut avoir été intervertie avec une autre). Pour en réinitialiser la détection, sous l'onglet Propriétés, cliquez sur Editer la configuration, puis Nouvelle auto-détection :

Dans le cas où vous ne comprenez l'anomalie à l'aide des informations affichées, réalisez une capture SNMP (à l'aide de SNMPWalker) permettant l'analyse du problème. Envoyez la capture SNMP en fournissant les informations suivantes :

  • Nom de l'anomalie,

  • Marque et modèle du périphérique,

  • Version du firmware,

  • Nom de la capture SNMP (aux formats .bin et .txt), ainsi qu'une capture d'écran (ou photo) des compteurs du périphérique (au format .jpg).
