Watchdoc - Droits d'accès et rôles d'administration

Rôles d'administration : détail

Le rôle donne le droit de :

- créer/éditer/supprimer des files d’impression

- créer/éditer/supprimer des groupes

- supprimer les documents en attente dans les files d’impression

- créer/éditer/supprimer des règles appliquées sur lesfiles

- créer/éditer/supprimer des tarifs

- créer/éditer/supprimerdes rôles.

- consulter les incidents (files)/consommables.

- créer/éditer/supprimer des Quota/Règles de gestion

- créer/éditer/supprimer des règles appliquées aux PMV

- créer/éditer/supprimer des rôles.

- recharger/supprimer un utilisateur.

Attention : ce droit ne donnant pas d'accès aux files d’impression , il est nécessaire de l'ajouter sur le groupe de files ou la /file lors de la création d'un quota.

- consulter les incidents (files)/consommables.

- consulter le bilan

- consulter l’historique

- consulter les statistiques.
- Consulter/supprimer les archives.

- consulter des files d’impression

- supprimer les documents en attente dans les files d’impression

- consulter les incidents sur les file liés aux consommables.

- recharger/supprimer un compte quota.

- créer/éditer/supprimer des tarifs.

- créer/éditer/supprimer des règles

- créer/éditer/supprimer des rôles.

- créer/éditer/supprimer des annuaires.

- créer/éditer/supprimer des filtres

- créer/éditer/supprimer des rôles.

- visualiser les documents en attente dans les files d’impression.

N.B. pour des raisons de respect de la confidentialité, ce rôle est rarement attribué.

- visualiser les données d'identification des utilisateurs.

A partir de la v. 6.1.4926, outre les nom et prénom, le code PUK est également considéré comme une identité utilisateur. Les opérateurs chargés de gérer les annuaires peuvent ainsi être autorisés, ou non, à voir le code PUK des utilisateurs.
