WDIS Watchdoc Data Integration Service - Utiliser

Gérer les anomalies avec WDIS

Anomalie lors de l’exécution du script

  • Symptôme : message d’erreur après l’exécution du script.

  • Analyse :

    • vérifiez la syntaxe de la commande ;

    • vérifiez la présence du .NET Framework ;

    • vérifiez le chemin d’accès WDIS.

Anomalie de réception

  • Symptôme : le document n’apparaît pas dans le répertoire souhaité ou aucun mail n’est reçu.

  • Analyse : vérifiez que vous avez bien spécifié le nom du fichier dans l’option « -file c:\WDreports\report1.csv » ou vérifiez dans Watchdoc que le serveur SMTP est bien renseigné et fonctionnel.

Anomalie de valeur remontée

  • Symptôme : lors de l’extraction sur des données en SNMP (compteur, consommables), vous constatez que certaines colonnes ne sont pas renseignées (exemple : sur les compteurs copies couleur).

  • Analyse : depuis l'interface web Watchdoc, vérifiez que l’information (consommable ou compteur) de la file est bien présente sur la file d’impression (onglet Etat).

 

  • Si l'information relative aux consommables et compteurs est présente :

    • vérifiez que l'argument [OPTIONS] ne comporte pas une valeur qui fausse le résultat (dates correspondant à une période d'activité de l'outil, notamment) ;

    • depuis l'interface web Watchdoc, vérifiez dans l’historique des incidents, que ce périphérique était bien contrôlé et disponible durant la période pour laquelle vous souhaitez obtenir des données ;

    • dans les fichiers de logs, vérifiez que Watchdoc a compressé les données SNMP (tâche planifiée par défaut à minuit, le logiciel Watchdoc compresse les datas SNMP, vérifiez la bonne réalisation de cette étape dans les logs.

  • Si l'information relative aux consommables n’apparaît pas :

     

  • vérifiez que le SNMP (v1) sur le périphérique d’impression ainsi que sur Watchdoc est bien activé.

  • Contacter le support Doxense en précisant les informations suivantes :

  • Nom de l’imprimante :
  • Marque :
  • Modèle :
  • Colonne vide ou fausse : (inscrire le nom figurant sur le tableau WDIS) :

  • Fournir les pièces jointes suivantes :

  1. Dumps SNMP (.bin et .txt) avec SNMPWalker sur l’adresse IP du périphérique d’impression (disponible avec la documentation sur le site de téléchargement de Doxense, ainsi que dans le sous-répertoire Tools du répertoire d’installation Watchdoc)

     

  2. Une capture d’écran (format .jpeg) des compteurs du périphérique d’impression (se rendre sur l’interface web du périphérique dans la partie Compteur) avec l’information sur le consommable ou le compteur manquant (dans notre exemple, il s'agit du compteur de copies couleur).
