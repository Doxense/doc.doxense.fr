Watchdoc - Base de données statistique Watchdocstats - Présentation

Principes

Dans Watchdoc, l'activité d'impression est enregistrée dans une base de données statistique appelée Watchdocstats.
Cette base est alimentée grâce aux données collectées par Watchdoc via le protocole SNMP Simple Network Management Protocol (Protocole Simple de Gestion de Réseau) est un protocole de communication qui permet aux administrateurs réseau de gérer les équipements du réseau, de superviser et de diagnostiquer des problèmes réseaux et matériels à distance. (Source : Wikipedia).

Elle peut-être utilisée à des fins d'analyse, mais aussi pour gérer les quotas d'impression.

Les données sont affichées dans l'interface d'administration de Watchdoc mais peuvent aussi être exploitées et exportées par WDIS Watchdoc Data Integration Services est un outil en ligne de commande, installé et fourni en standard avec Watchdoc, permettant de générer des rapports relatifs aux compteurs et aux consommables des files contrôlées..

Par ailleurs, si votre organisation ne dispose pas d'annuaire dédié (Active Directory), un annuaire peut être créé dans une base de données MS SQL®.

Enfin, c'est également sur une base MS SQL que repose l'annuaire Invités (base Guests) permettant de gérer ponctuellement des utilisateurs non présents dans l'annuaire de l'organisation.

Il est donc nécessaire d'installer une base de données préalablement à l'installation de Watchdoc.

 

Par ailleurs, dans la mesure où les données de la base varient, au fil du temps, il arrive que le traitement des requêtes soit ralenti en raison d'une fragmentation des index. Pour éviter cet inconvénient, nous vous recommandons de nettoyer régulièrement votre base en suivant le plan de maintenance fourni. 

Prérequis

Avant d'installer la base de données MS SQL, vérifiez les prérequis suivants :

  • la base de données n'est pas une base MS SQLExpress ;

  • la version de MS SQL est compatible avec la version de Watchdoc (cf. les prérequis de la version Watchdoc concernée) ;

  • l'outil MS SQL Management Service est installé sur le serveur (librement téléchargeable depuis le site Microsoft).

     

La base de données MS SQL peut être installée sur le même serveur que Watchdoc (mode autonome) ou sur un autre serveur (mode classique et mode déporté).

Dans le cas où le serveur SQL est distant, vérifiez le prérequis suivants :

  • TCP/IP est activé dans SQL Server Configuration Manager ;

  • le service SQL Browser est démarré afin que Watchdoc puisse afficher la liste des serveurs et les instances disponibles ;

  • le port TCP 1433 est ouvert sur le pare-feu pour l'instance ;

  • le port UDP 1434 est ouvert sur le pare-feu pour SLQ Server Browser.
