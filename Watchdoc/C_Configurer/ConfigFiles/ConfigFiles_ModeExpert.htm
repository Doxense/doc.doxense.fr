Titre

Par défaut, Watchdoc attribue au travail d'impression le titre qu'il récupère dans la balise titre (<title>) de l'en-tête (<header>) du fichier de spool (.shd ?). Dans certains cas, il peut être nécessaire de modifier ce fonctionnement par défaut et d'en appliquer un autre.

ignorer le titre extrait du spool : cochez cette case pour ne pas tenir compte du titre du document enregistré dans le fichier de spool. Décocher cette case est utile dans le cas où vous constatez des incohérences entre le titre du document et le titre affiché dans Watchdoc car il arrive en effet que certains pilotes renvoient à Watchdoc un titre erroné. Dans ce cas, préciser quel est le titre qui est attribué au document.

remplacer le titre des documents par un texte générique dans l'historique des impressions : cochez cette case pour que Watchdoc remplace le titre du document par un titre générique dans l'affichage de l'historique des impressions. Comment WD conmpose-t-il le titre générique ?

chiffrer le titre des documents dans la file d'attente partagée : cochez cette case si vous souhaitez que Watchdoc transforme en une suite aléatoire de chiffres le titre des documents, de sorte que leur titre ne soit pas lisible dans la file d'attente partagée. Pourquoi vouloir chiffrer le titre ? Dans quel cas est-ce utile ?

Copies

Ignorer le nombre de copies extrait du spool : cochez cette case dans quel cas faut-il

Corriger automatiquement si nombre de copies au carré : cochez cette case dans le cas où vous êtes confronté à un dysfontionnement généré par un pilote qui multiplie de manière exponentielle le nombre de copies.

Recto-verso : Ignorer le mode recto-verso extrait du spool. Cochez cette case pour désactiver le mode recto-verso et imposer le mode recto sur un périphérique.

N-to-Up : Ignorer le mode N-to-Up extrait du spool : cochez cette case dans le cas où le pilote gère incorrectement le mode N-to-Up (2 pages en 1, 4 pages en 1,par exemple), de manière à ignorer ce mode de fonctionnement ;

Fantômes : détecter et supprimer automatiquement la dernière page vide. Cochez cette case si vous souhaitez que Watchdoc détecte et n'imprime pas les pages vides des travaux d'impression (page ne comportant aucun caractère).

Identité Ignorer l'identité extraite du spool : cochez cette case pour ne pas tenir compte de l'identité de l'utilisateur renvoyé par le pilote dans le cas où l'identité est erronée. Dans ce cas, qui et comment est généré l'identité de l'utilisateur ?

Station

Ignorer le nom de station extrait du spool : cochez cette case pour ne pas tenir compte si vous souhaitez que le nom de la station (poste de travail) d'où sont lancés les travaux d'impression ne soit pas pris en par Watchdoc.

Utiliser la station comme nom d'utilisateur : cochez cette case si vous souhaitez que le nom de la station de travail (poste client) soit utilisé en guise d'identité utilisateur.

Domaine.  Forcer le domaine des utilisateurs de cette file : cochez cette case dans le cas où vous souhaitez désigner comme domaine un annuaire spécifique ou un annuaire proxy que vous choisissez dans la liste déroulante située sous la case à cocher. D'où viennent les mentions de la liste ? Sont-elles prédérifinies, varient-elles en fonction d'un autre param ?

Auth. Titre

Extraire l'identité du titre du document s'il correspond à : cochez cette case si l'identité de l'utilisateur apparaît dans le titre du fichier de spool et que vous souhaitez l'exploiter. Vous devez alors compléter la configuration en saisissant dans la zone suivante une expression régulière Les expressions régulières (également appelées Regex ou Regexp) sont un système puissant et rapide pour mener des recherches dans des chaînes de caractères informatiques. Elles reposent sur une syntaxe spécifique utilisant des caractères ordinaires alphanumériques ainsi que des caractères spéciaux appelés "métacaractères". Les expressions régulières utilisées dans Watchdoc sont conformes au DotNet Framework. Pour concevoir vos expressions régulières, référez-vous aux sites et aides en ligne relevant de ce standard. (regular expression) indiquant la manière d'extraire la chaîne de caractères correspondat à l'identité de l'utilisateur, par exemple : "^GenericApp - User:([A-Za-z0-9_]+)$"
