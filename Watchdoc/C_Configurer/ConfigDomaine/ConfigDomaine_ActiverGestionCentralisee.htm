Watchdoc - Activer la configuration centralisée / réplication

Principe

Watchdoc permet de synchroniser différents éléments de configuration entre le serveur master et les autres serveurs qui en dépendent. 

Cette réplication est activable pour les éléments de configuration suivants :

  • Files d'impression

  • Tarifs

  • Annuaires

  • Filtres

  • Notifications

  • Profils de sécurité

  • Profils de déblocage

  • Télémétrie (communication des données à WSC - Console de Supervision)

 

Une fois activée, la configuration centralisée ne peut pas être désactivée. 

Activer la réplication

Pour activer la réplication d'un élément de configuration depuis le serveur master :

  1. dans l'interface d'administration Watchdoc du serveur master, sélectionnez l'élément de configuration à répliquer ;

  2. cliquez sur le bouton Editer les propriétés de l'élément de configuration à répliquer (groupe de files et annuaires dans l'exemple) ;

  3. dans la section Informations générales, cochez la case Global ;

  4. cliquez sur Valider pour enregistrer la configuration :

è Une fois le paramètre déclaré comme Global, il est répliqué sur tous les serveurs Watchdoc et toute modification effectuée sur le serveur master est automatiquement répliquée sur tous les autres serveurs (le délai de réplication dépend du mode de synchronisation appliqué dans le domaine, cf. Configurer le mode de synchronisation).

Une fois cochée, la case Global ne peut pas être décochée.

Sur le serveur master, le logo signale les éléments de configuration répliqués sur les autres serveurs.

Sur les serveurs du domaine, le logo précise que la configuration provient du serveur master.
