Watchdoc - Emplacements - Présentation

Principe

Dans Watchdoc, un emplacement est un libellé appartenant à une liste structurée de sites géographiques. Il permet de localiser physiquement des éléments d'un parc d'impression (périphériques d'impression, serveurs, files, utilisateurs, etc.). Il existe différents types d'emplacements correspondant à différents niveaux hiérarchiques (de Monde à Site).

 

Par ailleurs, les emplacements permettent de détecter les serveurs et files d'impression disponibles pour un utilisateur de WPC en fonction des adresses IP du Print Client utilisé et des éléments du parc d'impression. 

 

Les emplacements sont administrables depuis l'interface d'administration de Watchdoc.

Dans un domaine (master/slaves), les emplacements sont gérés depuis le serveur master et non depuis les slaves. Dans les serveurs slaves, l'administreur peut consulter la liste des emplacements.

 

L'emplacement Monde est la localisation par défaut dont dépendent tous les autres emplacements. Les emplacements doivent être créés sous celui-ci.

Prérequis

  • Lors de la configuration initiale, vous pouvez choisir de ne pas utiliser les emplacements si vous souhaitez utiliser Watchdoc sans les Print Clients.

  • En revanche, vous devez impérativement les configurer pour utiliser un ou les Watchdoc Print Clients suivants :

    • WPC for Android

    • WPC for Chrome

    • WPC for Windows

    • Skyprint

  • Vous devez connaître l'infrastructure réseau, notamment des plages d'adresses IP, pour configurer les emplacements et surtout le niveau "site".

  • Le serveur doit être associé à un emplacement pour être en mesure de router les travaux d'impression vers les périphériques d'impression à l'aide des Print clients. Si un serveur figure dans l'emplacement Indéterminé, il n'est pas en mesure de traiter les demandes d'impression adressées par les Print Clients.

  • Le site, dernier emplacement de la hiérarchie, est associé à une liste d'adresse IP de sous-réseaux afin de détecter automatiquement les périphériques d'impression connectés à un serveur. De même, grâce à l'adresse IP de connexion des postes, Watchdoc peut localiser automatiquement les utilisateurs qui utilisent Watchdoc Print Client.

Principes de routage

  • Quand un utilisateur choisit un emplacement, il obtient la liste des emplacements de type "Sites" auxquels sont associés les périphériques d'impression.

  • Si un serveur est associé à ce Site (ou si ce Site dépend d'un parent associé à un serveur), ce serveur traite la demande d'impression.

  • En revanche, si le site n'est associé à aucun serveur (ou s'il dépend d'un parent qui n'est associé à aucun serveur), aucun serveur ne traite la demande d'impression. Dans ce cas, il est donc possible que l'utilisateur voie des périphériques d'impression associés au site mais ne puisse pas imprimer. D'où l'importance d'associer un serveur d'impression à tout site (ou parent du site).

  • De plus, un routage peut être envisagé entre plusieurs serveurs. Dans ce cas, la route doit partir du serveur master pour aller vers un serveur slave. Dans le sens inverse, le routage ne fonctionne pas.
