Watchdoc Mobile Client - Ajouter et configurer l'application dans l'EMM Intune

Principe

En tant qu'administrateur, lorsque vous fournissez votre ID d'organisation, Doxense l'ajoute à la liste des organisations / entreprises autorisées à utiliser l'application Watchdoc Mobile Client. Cette application est mise à votre disposition dans le store Google Play. 

 

Dès que vous avez accès à l'application dans Google Play, vous pouvez la mettre à disposition de vos utilisateurs de deux manières différentes :

  1. soit en la configurant dans votre EMM Microsoft Intune de sorte qu'elle soit mise à disposition de vos utilisateurs depuis leur environnement Intune professionnel ;

  2. soit en fournissant à vos utilisateurs l'application sous forme d'un fichier .apk. Chaque utilisateur doit installer le fichier .apk sur son smartphone, puis doit le configurer à l'aide des paramètres que vous fournissez.


Les paramètres à configurer ou à fournir à vos utilisateurs, sont :

  • l'url du "serveur de découverte" qui permet à l'application de communiquer avec le serveur d'impression Watchdoc ;

  • le certificat de la Print API (fichier au format .der), garant de la sécurité de l'application.

Prérequis

Pour configurer l'application Watchdoc Mobile Client dans Intune, vous devez disposer du certificat print_api_cert.der au format base 64.
Ce certificat doit être exporté au format .der depuis Watchdoc puis modifié en fichier .txt :

  1. connectez-vous à l'interface d'administration de Watchdoc ;

  2. depuis le Menu Principal, section Configuration, cliquez sur Configuration avancée > Configuration Système ;

  3. dans la section Divers, cliquez sur Exporter le certificat :

  1. enregistrez le fichier print_api_cert.der généré dans votre espace de travail ;

  2. utilisez ensuite l'outil en ligne de commande Certutil.exe (cf. documentation certutil) pour obtenir un certificat au format .txt :

     

  3. Enregistrez le fichier print_api_cert.txt dans votre espace de travail : vous en aurez besoin pour configurer l'application Watchdoc Mobile Client dans Intune.

Configurer Watchdoc Mobile Client dans InTune

Pour que l'application Watchdoc Mobile Client puisse communiquer avec le serveur Watchdoc, il convient d'associer l'application Watchdoc Mobile Client à l'url du serveur d'impression dans l'EMM et de l'associer au certificat :

  1. rendez-vous dans l'interface d'administration ;

  2. dans l'interface de gestion des applications, accédez à la liste des applications (Accueil > Applications) ;

  3. sélectionnez ensuite l'entrée iOS/iPadOS ou Android en fonction du système d'exploitation sur lequel l'application doit être installée ;

  4. dans Managed Google Play, cherchez et sélectionnez l'application Watchdoc Mobile ;

     

  5. dans l'interface de gestion de l'application, cliquez sur Propriétés (Properties) ;

  6. créez la politique de déploiement de l'application ;

  7. sous l'onglet Basics, indiquez :

    1. dans le champ Nom de l'application (Watchdoc Mobile Client, par exemple) ;

    2. dans le champ Platform, le système d'exploitation pour lequel l'application est configurée

    3. dans le champ Type de profil, sélectionnez le type "fully managed, dedicated and Corporate-Owned Work Profile Only" ;

  8. cliquez ensuite sur le bouton Next pour accéder à l'onglet Settings

Configurer l'url du serveur Watchdoc dans l'application

  1. Sous l'onglet Settings, sélectionnez le format Use configuration designer ;

  2. Cliquez ensur le bouton +Add pour ajouter deux clés de configuration :

    • Clé de configuration 1

      • saisissez l'identifiant de la clé ("Serveur de découverte" ou "Serveur Watchdoc", par exemple :

      • Value type : sélectionnez la valeur string ;

      • Configuration value : saisissez l'adresse url du serveur master sur lequel est installé Watchdoc ;

      • Description : saisissez un texte indiquant la nature de la clé de configuration ;

    • Clé de configuration 2

      • saisissez la mention "Certificat de licence" ;

      • Value type : sélectionnez la valeur string ;

      • Configuration value : ajoutez le certificat (format base 64). 
        Pour obtenir le certificat en base 64, reportez-vous à la section Prérequis ci-dessus ;

  3. Dans la section Connected apps, cliquez sur Enabled pour autoriser vos utilisateurs à se connecter à l'application à l'aide de leur profil personnel ou professionnel.

  4. Cliquez sur Next pour enregistrer les paramètres.
