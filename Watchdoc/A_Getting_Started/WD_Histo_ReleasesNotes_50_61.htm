Watchdoc 6.1.x

Watchdoc/WSC 6.1.1.5533, WPC for Windows et Skyprint 7.0.5979 - 23/02/2026

Correctif

  • WES Xerox :

    • correction d'une anomalie qui empêchait la création du WES Xerox lorsque la case "comptabilisation par le WES (WES accounting) était cochée".

Watchdoc/WSC 6.1.1.5528, WPC for Windows et Skyprint 7.0.5979 - 18/02/2026

Améliorations

  • Watchdoc :
    • dans la configuration des annuaires SQL, suppression du type "Oracle" incompatible avec Watchdoc
    • dans la gestion des règles sur les files, suppression de la possibilité de rediriger un travail d'impression vers un pool de périphériques d'impression dans la mesure où ce fonctionnement n'est pas supporté par Watchdoc
    • dans l'interface de gestion d'un domaine, la liste des serveurs slaves est désormais paginée pour en faciliter la gestion
    • harmonisation du paramètre "Durée de vie" par défaut (8heures) sur tous les types de files (universelles, physiques et virtuelles)
    • dans l'interface de consultation des statistiques (administration), rétablissement du bouton Exporter
    • dans la gestion d'une file d'impression, remplacement du champ de saisie par des cases à cocher pour le paramètre Tag (section Plateforme)
  • WSC :
    • lors de la propagation des packages de mise à jour, clarification du message d'erreur "Security check failed" causé par une absence de droits d'écriture et de lecture sur les dossiers de stockage
    • dans l'interface de configuration du serveur, il n'est plus possible de configurer WSC avec un port non-sécurisé (5757), configuration qui générait des messages d'erreur "BadRequest"
    • clarification des libellés et messages d'erreurs dans les interfaces d'import de fichiers .CVS
  • WPC for Windows :

    • correction du port pris en compte par le service de découverte (discover) pour privilégier le port sécurisé (5753 par défaut) au lieu d'un port non-sécurisé

    • lors du changement du template d'une file universelle (depuis le serveur master), répercussion instantanée sur les WPC installés sur les postes clients

  • WEScan : lors de la configuration d'un profil WES, il est désormais impossible de valider le profil si aucun profil de numérisation n'est sélectionné alors que la fonction Numérisation est activée

 

Correctifs

  • Watchdoc :
    • dans les notifications par mail, il arrivait que l'heure indiquée soit erronée (décalage d'une heure). Désormais, la notification s'ajuste à l'heure locale
    • dans la page "Mon compte", affichage de la prévisualisation de tous les travaux d'impression, même dans le cas où le chiffrement est activé sur Watchdoc, ce qui n'était pas possible dans les versions précédentes
    • correction d'une anomalie d'affichage du code PUK dans la page "Mon compte" dans le cas où l'utilisateur ne dispose pas de code PIN
  • WPC for Windows : correction d'une anomalie qui empêchait le traitement de documents volumineux (> 150 pages). Lorsque l'utilisateur lançait une demande d'impression, le document apparaissait à l'état "Envoyé", figurait dans la liste des travaux sur le WES mais ne pouvait pas être imprimé
  • WEScan : modification du composant gérant l'interaction entre la JSON DB et SQL pour éviter la suppression de numérisations lors de l'activation de la fonction ScanToFolder

  • WSC/PNP :

    • lors de la migration d'une file d'un serveur à l'autre à l'aide de Plug and Print, un message indiquait "Migration en échec : the queue of id [N°_ID] already exists!" lorsqu'une file de type proxy existait. Désormais, la file proxy est supprimée avant le transfert

    • il arrivait que le numéro de version inscrit dans le fichier de sauvegarde soit erroné. Désormais, le bon numéro de version est inscrit dans la gestion des comptes Invités,

    • contrôle du champ Email rendant cette valeur facultative et permettant d'éviter les doublons (adresse email attribuée à deux invités différents)

  • WES Sharp:

    • prise en compte de la langue configurée sur le périphérique pour afficher les messages du WES lorsque l'authentification par badge est activée

    • réactivation du logo comme bouton d'action pour accéder aux actions spéciales sur l'écran du périphérique lorsque l'authentification par badge est activée

Watchdoc/WSC 6.1.1.5514, WPC for Windows 7.0.5973 et Skyprint 7.0.5973 - 04/02/2026

Améliorations

  • Watchdoc
    • lors de la configuration d'un événement de type "redirection vers..." envoyant l'impression vers une file physique, amélioration de l'information pour prévenir que l'impression risque d'être effectuée sans validation
    • ajout d'une aide en ligne dans la configuration des emplacements avec prérequis et principes de routage et message d'alerte en cas de configuration erronée ou incomplète
  • WPC for Windows

    • les files d'impression supprimées dans Watchdoc sont désormais également supprimées dans le gestionnaire d'impression MS Windows

    • lorsqu'un utilisateur définit une file par défaut, elle s'affiche désormais au début de la liste des périphériques disponibles
  • WES Hewlett Packard : lors de l'installation du WES, amélioration des messages relatifs au nettoyage des certificats : l'administrateur sait désormais pour quelle raison l'action a réussi ou échoué

  • WEScan : amélioration des informations figurant dans les fichiers traces dans le cas où le profil de scan n'a pas été trouvé. Les informations présentes dans les logs permettent désormais un meilleur diagnostic en cas de dysfonctionnement

  • Skyprint : meilleur traçage des opérations de configuration : les modifications et suppressions de configuration sont désormais présentes dans les fichiers de logs afin de faciliter le diagnostic en cas de dysfonctionnement

 

Correctifs

  • Watchdoc : correction d'une anomalie survenant lors de la suppression d'une source de données et qui en empêchait la suppression

  • WSC : correction d'une erreur qui survenait au lancement de l'assistant de configuration d'un rapport (Configuration avancée > Envois de mails > Rapport...). Le message "Method Not Allowed" s'affichait à la place de l'interface "Configuration d'événement périodique" 

  • WES

    • Lexmark : suppression d'un message d'erreur dans le rapport d'installation (pull print app is not installed) alors que le WES est correctement installé ;

    • Sharp : prise en compte des caractères accentués et diacritiques dans le nom des applications affichées sur le WES et dans les interfaces d'administration des périphériques.

 

Watchdoc/WSC 6.1.1.5500, WPC for Windows 7.0.5957 et Skyprint 7.0.5957 - 21/01/2026

Amélioration

  • Watchdoc : suppression de certaines couleurs (jaune, notamment) qui rendaient difficile la lecture des initiales dans les identicônes

  • WPC for Windows :

    • amélioration des performances au démarrage du service pour limiter le nombre de requêtes ICMP adressées au serveur master

    • amélioration de l'interface de sélection de l'emplacement afin d'éviter le débordement de certains boutons lorsque la taille de l'interface était réduite

  • WES

    • Lexmark : lors de l'installation du WES sur la file, clarification du rapport indiquant que le mot de passe administrateur est erroné ou qu'il a été oublié (“Could not retrieve session Id, probably because security is enabled on the device" devient "Could not retrieve session Id, please check the password")

    • Sharp : dans la configuration du profil WES, ajout d'un contrôle sur des paramètres de la section Gestion de la redirection afin d'éviter une erreur de configuration

    • Xerox : dans la configuration du profil WES, ajout de messages expliquant la dépendance entre le mode Comptabilisation à partir du périphérique et les Quotas afin d'éviter une erreur lors de l'enregistrement du profil

  • WEScan - ScanToMail : ajout d'informations dans les logs (notamment informations WARN) lorsque la fonction ScanToMail échoue (unsuccessfully sent) et que le scan est supprimé. Ces logs complètent les informations affichées dans l'interface des notifications Watchdoc

  • WPC for Android : si l'organisation ne dispose pas d'EMM pour mettre l'application à disposition des utilisateurs, il est possible de l'installer à partir d'un .apk

 

Correctifs

  • Watchdoc :

    • correction de la gestion des proxy inverses dans l'interface de gestion Serveur web > DSP permettant l'ajout et la suppression effectifs des adresses saisie

    • correction du processus de redirection d'un travail d'impression pour éviter de générer des spools non-traités et nettoyage automatique de ces spools non-traités afin de limiter l'encombrement du spooler

    • contrôle des codes PIN saisis par l'utilisateur dans la page "Mon compte" : désormais, la longueur du code respecte la longueur attendue

  • WSC : la désactivation de l'obfuscation des mots de passe est désormais opérationelle

  • WES Toshiba : correction de l'installation du WES Toshiba pour rendre la fonction ScanToMe opérationnelle. Il arrivait que l'installation du WES désactive l'annuaire LDAP configuré sur les périphériques d'impression

  • WEScan Kyocera : suppression de caractères de ponctuation qui s'affichaient dans les options d'orientation des travaux et ajout de traductions manquantes

Watchdoc/WSC 6.1.1.5486, WPC for Windows 7.0.5944 et Skyprint 7.0.5947 - 07/01/2026

Amélioration

  • WPC for Windows : l'utilisateur qui ne souhaite pas utiliser WPC n'est plus obligé de s'authentifier au démarrage du service (l'interface d'authentification s'affichait avec insistance jusqu'à l'authentification). Il est alors informé qu'il devra s'authentifier s'il souhaite imprimer. Lors d'un changement de session, l'interface d'authentification s'affiche au nouvel utilisateur après démarrage du service.

  • Skyprint :

    • amélioration de la configuration des conditions d'acceptation des emails. Il est ainsi plus facile d'utiliser les restrictions par défaut ou les restrictions spécifiques à une boîte mail. De plus, il est possible de supprimer des domaines ou adresses mails précédemment autorisés ;

    • amélioration de la gestion des reconnexion entre Skyprint et les boîtes mails supervisées en cas de redémarrage suite à une panne.

  • WES Lexmark : dans le profil WES, ajout d'un paramètre permettant d'activer la suppression des documents suspendus.

Correctifs

  • Watchdoc

    • Mise à jour des composants gérant la communication avec une base de données Postgre SQL afin d'éviter un dépassement du nombre de connexions simultanées autorisées. Ce dépassement générait le message d'erreur "Internal error while connecting to database pgsql..."

    • La base Cards (Badges) n'était pas automatiquement créée dans WSC lorsque la base de données configurée était de type Postgre SQL. L'anomalie est désormais corrigée.

    • Correction de la dépendance de Watchdoc avec une base SQL : lorsque la connexion avec une base SQL n'était pas établie, il arrivait que Watchdoc ne démarre pas. Désormais, Watchdoc démarre, que la connexion avec la base soit établie ou non.

    • Lorsque Watchdoc fonctionnait avec le spouler Doxense (custom spooler), il arrivait qu'un quota soit débité deux fois si le travail était relancé après incident d'impression. Désormais, le quota n'est débité qu'une seule fois. S'il y a eu deux débits, le second est remboursé.

  • WES Sharp : correction du dimensionnement des icones "Mes impressions" et "WEScan" sur l'écran du périphérique. Il arrivait que la taille des icones diffère. Les icones sont désormais de même taille (cf. Dépanner le WES Sharp).
