Capturer les spools Watchdoc

Pour capturer le spool afin d'analyser une anomalie reproductible, il convient tout d'abord de "bloquer" ce spool.

Méthode simple

  1. rendez-vous sur le serveur d’impression ;

  2. dans le gestionnaire d'impression, cliquez-droit sur la file à analyser ;

  3. dans la liste des actions, cliquez-droit et sélectionnez "suspendre" ;

  4. mettez la file d’impression à analyser en pause.

 

Méthode exploitant un filtre

  1. Authentifiez-vous dans l'interface d'administration de Watchdoc® en tant qu'administrateur;

  2. depuis le Menu Principal > section Gestion, cliquez sur Filtres :

  1. dans la liste des filtres disponibles, cliquez sur le bouton Créer un nouveau filtre ;

  2. dans l'interface Création d'un filtre, en section Filtre, remplissez le champ Nom avec un nom explicite (par exemple : "BloquerImpressions") ;

  3. Paramétrez une Règle permettant de rejeter les impression provenant d'un identifiant d'utilisateur prédéfini habilité à procéder au test :

  4. en Condition, sélectionnez "Utilisateur" > "Le nom de l'utilisateur est"... ;

  5. dans le champ Paramètre, saisissez l'identifiant de l'utilisateur habilité à mener le test (logiquement, vous pouvez utiliser le vôtre) ;

    • Si vrai, activez l'action "Sélectionner" ;

    • Si faux, activer l'action "Rejeter".

  1. Cliquez sur le bouton Créer pour valider le filtre.

  2. Revenez au Menu principal puis, dans la section Exploitation, cliquez sur Files d'impression ;

  3. dans la liste des files, sélectionnez la file que vous souhaitez analyser ;

  4. dans l'interface de gestion de la file, cliquez sur le bouton Règles :

  1. dans l'onglet Evènements, cliquez sur le bouton Editer de la file d'impression :

  1. dans la liste Evènements sur la file, sélectionnez le filtre précédemment créé, puis appliquez l'action "Bloquer le document" ;

  2. cliquez sur le bouton pour valider l'ajout du filtre permettant de bloquer les spools sur cette file.


Si, après analyse du fichier de spools, l'équipe Support Doxense® doit prendre la main sur le serveur afin de poursuivre le diagnostic, nous vous recommandons d’installer l’outil gratuit Team Viewer® qui permet une prise en main à distance sur votre serveur.
