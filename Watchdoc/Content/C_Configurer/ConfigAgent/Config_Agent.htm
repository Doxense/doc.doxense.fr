Configurer l'Agent Watchdoc (Agent local)

Configurer l'Agent Watchdoc pour un poste

  1. Dans l'interface de configuration de l'Agent Watchdoc :

  2. dans l'onglet Connection Settings, sélectionnez le protocole utilisé pour permettre à l'Agent de communiquer avec Watchdoc :

    • soit Direc.NET Remoting ;

    • soit HTTP XML RPC (pour accéder au site IIS).

  1. en fonction du choix du protocole, indiquez ensuite :

    • le nom DNS du serveur Watchdoc ou du serveur web (IIS) de Watchdoc ;

    • l'URI ou l'URL du serveur ;

    • le numéro du port (par défaut, 5744 pour Direct.Net Remoting et 80 pour le protocole HTTP XML RPC) ;

    • pour le protocole HTTP XML RPC, l'identifiant du serveur.

  1. Indiquez dans la section Schedule le mode selon lequel les travaux d'impression doivent être envoyés au serveur Watchdoc :

    • cochez la case Real time pour que les travaux soient envoyés en mode synchrone ;

    • cochez la case Only send job data every pour que les travaux soient envoyés en mode asynchrone et choisissez dans la liste la fréquence d'envoi des travaux d'impression vers le serveur Watchdoc.

  1. Cliquez sur le bouton Apply pour enregistrer la configuration puis cliquez sur Exit pour quitter l'interface.
