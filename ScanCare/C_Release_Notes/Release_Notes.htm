Version 9.10.705 - 29/04/2025

Corrections

Correction du fait que la sortie Teams ne fonctionnait pas sur les profils SPL.

Correction du fait que les fichiers n'étaient pas supprimés après un processus de sortie réussi avec un compte réseau.Dans certaines circonstances, les documents n'étaient sont pas téléchargés dans ScanCare.

Version 9.10.625 - 29/01/2025

Améliorations

  • Message d'avertissement lors de l'ouverture de l'interface d'administration si la limite de pages de la licence SPL est atteinte

Corrections

  • Dans certaines circonstances, les documents n'étaient sont pas téléchargés dans ScanCare

  • ScanCare SPL/UDL : les PDF entrants étaient sauvegardés alors qu'aucun traitement n'était activé.

Version 9.10.615 - 29/11/2024

Améliorations

  • Prise en charge de la version

Version 9.10.605 - 24/10/2024

Nouvelle fonction

  • ajout de PDF/A 1a et PDF/A 1b dans les options de conversion.

Corrections

  • Sharp Luna Series - Correction d'un problème qui faisait que Luna ne voyait que la bannière et un fond blanc.

  • Lorsque la bibliothèque SharePoint était modifiée sur l'appareil, elle n'était modifiée qu'à l'écran. Par conséquent, le document était stocké dans la bibliothèque configurée. Désormais, la sélection sur l'appareil est appliquée correctement et le document est stocké dans le dossier sélectionné.

Version 9.10.565 - 02/09/2024

Nouvelle fonction

  • Mettre à jour les composants de Drivve DM dans Drivve Image.

 

Corrections

  • Le profil dupliqué n'avait pas les paramètres d'analyse du profil original.

  • Correction d'un problème qui faisait que ScanCare perdait ses licences.

  • La source de données XML n'était pas synchronisée.

Version 9.10.525 - 18/03/2024

Corrections

  • Le champ de saisie des métadonnées géré par Sharepoint ne fonctionnait pas.

  • Lors de l'ajout manuel d'un dispositif, l'URL du serveur n'était pas définie et il n'était pas possible d'appuyer sur le bouton OK.

Version 9.10.505 - 23/11/2023

  • Les appareils Sharp OSA3 ne pouvaient pas se connecter à ScanCare.

  • Les appareils Toshiba ne numérisaient qu'une seule page si la numérisation en niveaux de gris était utilisée.

  • Les paramètres de résolution dans les préférences IRIS ne fonctionnaient pas comme prévu.

  • Ajout du support de Edge pour le Image simulator

Version 9.10.275 - 10/08/2022

Améliorations

  • Ajout de la prise en charge du distributeur Azure App et GraphApi Email. Nouvelle source de données disponible : Office 365/Azure

  • Le filtre Barcode offre de nouvelles options : "Le code-barres ne commence pas par", "Le code-barre ne finit pas par...".

  • Restriction de l'utilisation du simulateur pour des raisons de sécurité.

  • Lexmark ESF3 : stabilisation du déploiement sur les anciens appareils Lexmark.

  • Amélioration de la sortie des emails SMTP.

 

Version 9.8.265 - Mai 2022

Améliorations

  • HP - Changement des paramètres de compression par défaut du scanner du MFP à "élevé".

  • Le simulateur est maintenant disponible, s'il n'y a que des clients de bureau ScanCare enregistrés.

  • FUJITSU - La deuxième requête basée sur les résultats d'une première recherche fonctionne sur les appareils FUJITSU pris en charge.

Corrections d'erreurs

  • Amélioration de la qualité des images après la conversion avec IRIS.

  • Le dossier de service, qui était hors service, ne pouvait pas être supprimé.

  • ScanCare convertissait les PDF en niveaux de gris en PDF monochromes.

  • Les utilisateurs et groupes AD ne s'affichaient pas correctement dans la fenêtre des autorisations.

  • Un message d'erreur est maintenant affiché, lors du choix d'un dossier One Drive avec trop de sous-dossiers.

Version 9.8.240 - Mars 2022

Nouvelles fonctionnalités

  • Ajout d'une expression régulière comme option de filtre pour la séparation des codes barres.

  • Ajout d'une option pour accéder aux informations sur les groupes d'utilisateurs connectés.

  • Ajout de la taille du fichier comme une variable.

Améliorations

  • Ajout de la fonction "Inverser l'ordre" dans tous les champs.

  • Ajout d'une option permettant de modifier les valeurs de la marge dans le cadre de la suppression des pages blanches.

  • Ajout de l'option "Rendu PDF avec augmentation du DPI" dans les paramètres de reconnaissance des codes barres.

  • Prise en charge de l'impression depuis Microsoft Office 365 / Azure OneDrive et Teams.

  • Ajout d'un nouveau paramètre de numérisation pour les appareils Konica Minolta "Original mixte".

Corrections d'erreurs

  • Les documents utilisant le format IRIS présentaient des couleurs inversées.

  • Fonctions de texte : Les crochets de fermeture des fonctions étaient ignorés.

  • Dans certains cas, des codes-barres spécifiques n'étaient pas reconnus.

  • Les utilisateurs et groupes AD ne s'affichaient pas correctement dans la fenêtre des autorisations.

  • Application Admin : les paramètres de conversion avancés pour les pdf ne s'affichaient pas avec un nouveau profil de conversion.

Version 9.6.485 - Mars 2022

Améliorations

  • Watchdoc ScanCare peut se connecter à Drivve DM via HTTPS(SSL).

  • Le nom de fichier est utilisé, lors de l'importation d'une source de données.

  • Ajout d'informations supplémentaires à la fenêtre de sélection des modèles Xerox.

Corrections d'erreurs

  • Correction d'une erreur non gérée lors de l'ajout de l'application Watchdoc ScanCare à un périphérique Xerox.

  • Dans certains cas, il n'était pas possible d'enregistrer un nouveau client sur le serveur de licences.

  • Dans certaines circonstances, les dossiers créés, qui étaient nécessaires pour les processus de numérisation dans le dossier Work/Service, ne pouvaient pas être supprimés et empêchaient toute action ultérieure.

Version 9.6.450 - Janvier 2022

Amélioration

  • Modification du délai d'attente dans le manager Output.

Corrections d'erreurs

  • correction d'affichage par UDL d'une valeur erronée UDL pour la page disponible.

  • correction de la sélection du dossier racine dans ScanCare via Azure (OneDrive/Teams) qui, dans certains cas, ne fonctionnait pas avec un serveur 2016.

  • Correction de l'erreur lors de la sortie d'impression lors de laquelle tous les fichiers entrants sur le profil UDL n'étaient pas imprimés.

  • correction de l'option "fusionner automatiquement les pages en fonction du code-barres" qui ne fonctionnait pas comme prévu.

  • correction de la fonction d'ajout qui, dans certaines circonstances, créait un message d'erreur dans le journal du processeur d'images.

     

Version 9.6.370 - Novembre 2021

Nouvelles fonctions

  • Hotfolder avec entrée de méta-fichier : ScanCare ignore désormais le fichier d'entrée jusqu'à ce que le méta-fichier soit disponible.

  • ScanCare supporte maintenant les nouveaux appareils Ricoh.

Améliorations

  • Après mise à jour réussi, le dossier de mise à jour en ligne est maintenant nettoyé.

Corrections d'erreurs

  • Dans certains cas, ScanCare sautait des enregistrements dans le fichier de sortie CSV.

  • ScanCare n'enregistrait pas les paramètres dans la liste des périphériques si tous les périphériques étaient désactivés.

  • Dans certaines circonstances, ScanCare insérait la valeur du champ de saisie de l'objet du courrier plusieurs fois dans l'objet en fonction du nombre de pièces jointes.

  • Correction du fait que le processeur de tâches se bloquait lors de la conversion au format de fichier Word et de l'utilisation d'IRIS.

  • Dans certains cas, si l'option "Exporter (mot de passe)" n'était pas cochée lors de l'exportation de la source de données, une erreur se produisait.

Version 9.2.205 - Mai 2021

Nouvelle fonction

  • UserPrincipalName est disponible dans la liste des variables AD et sera rempli si un utilisateur est connecté.

Amélioration

  • CustomConfig - Dans un profil, il est maintenant possible de modifier les propriétés de plus d'un appareil pour le même type dans le fichier CustomConfig.xml.

  • Konica Minolta - Ajout d'une option permettant de définir une valeur pour la suppression de l'arrière-plan dans les préférences du scanner.

  • Ajout de la prise en charge du code à barres IATA 2 of 5

Corrections d'erreurs

  • Dans certains cas, l'outil IIS Config affiche un message de réussite, même si un message d'erreur apparaît.

  • La séparation des codes à barres se comporte différemment lorsque la reconnaissance des codes à barres ordinaires et la fonction de codes à barres attendus par page sont activées.*

  • Dans certains cas, la fenêtre Variables et Fonctions est vide dans la sortie ROUTE.*

  • Correction de la fonction de test de la source de données universelle

  • Dans certains cas, {FAX_TO} s'interrompt lorsque <ADS:AccountName> a été renseigné

  • Dans certains cas, la mise à jour en ligne de ScanCare n'arrêtait pas le service ScanCare Jobs / ScanCare Conversion pendant la mise à jour

  • Dans certains cas, la conversion IRIS Word a entraîné un document erroné.

Janvier 2021

Améliorations

  • Sortie ODBC : des variables supplémentaires ont été ajoutées dans la section "Ajouter des fonctions et des variables" pour étendre les instructions SQL

  • Mise en place d'un système de surveillance ScanCare Service

  • Dispositif UDL : Visibilité optimisée pour le bouton Test et peut être ajouté en tant que dispositif DEMO

  • Permettre la configuration des dimensions minimales des images .pdf pour le traitement des images

  • Imprimante LPR : il n'est pas possible d'enregistrer une imprimante LPR sans sélectionner un port de fichier et une étiquette indique que le LPR ne fonctionne qu'avec un port de fichier (*.prn)

  • Watchdoc ScanCare® nettoie les fichiers stockés dans le dossier temporaire de Windows

  • Ajout d'une "zone de silence" pour la reconnaissance des codes-barres

  • La fonction "Auto Color" n'est plus réinitialisée après le passage en mode duplex sur le MFP

  • Source de données universelle : options de courrier électronique et de télécopie ajoutées

  • Optimisation du réglage des fonctions "Eroder" et "Dilater" pour la reconnaissance des codes-barres

  • L'option de sortie de fichier "fichier joint", affiche un message d'erreur pour un format de conversion incompatible

  • Changement du message de confirmation après contrôle de la source de données Azure

  • Champ "sortie-entrée de fichier" avec expression régulière utilisé comme partie du nom du fichier

Corrections d'erreurs

  • LDAP : message d'erreur créé pour signaler le bases de données LDAP trop lourdes

  • Watchdoc ScanCare® ne pouvait pas lire les codes-barres QR de certaines images

  • Xerox : dans certaines circonstances, Watchdoc ScanCare® ne montre pas les modèles des clients

  • Champs de saisie : le curseur saute au début lors de la suppression de caractères successifs après un espace en utilisant le retour arrière

  • Dans certains cas, aucun résultat OCR zonal n'a été écrit dans le XML de sortie

  • Conversion de fichiers : le type de PDF "Texte" dans les paramètres PDF d'IRIS a causé une erreur

  • Dans certains cas, il n'a pas été possible d'extraire le texte du PDF consultable
