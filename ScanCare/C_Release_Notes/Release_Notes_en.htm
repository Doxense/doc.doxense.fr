Watchdoc ScanCare Watchdoc ScanCare 9.10.770

Version 9.10.760 - 09/03/2026

Enhancements

  • Warning message when unauthorized characters are entered in the name field

  • Advanced logging for UDL FTP entries

  • Addition of a new Lexmark add-on module

Error Corrections

  • Using a backreference in a regular expression causes an error

  • Not all attachments have the specified filename when splitting is enabled

  • In some cases, it was not possible to select multiple languages in the OCR conversion settings

  • The “Crop” checkbox does not work when selected in the OCR splitting settings

  • Attachments whose names contain the # character are not sent if the “Send a separate email per document” option is enabled

Version 9.10.735 - 01/12/ 2025

Error Corrections

  • File output: The “Run application” operation was performed only once, not for each document in the output

Version 9.10.705 - 29/04/2025

Error Corrections

  • Fixed that Teams output did not work on Server Processing License profiles.

  • Fixed that SSO checkbox was not displayed although a license exists.

Version 9.10.565 - 29/01/2025

Enhancements

  • Warning message when opening the admin UI if the SPL license page limit is reached

Error Corrections

  • Under certain circumstances documents are not uploaded to Drivve | DM version 11

  • ScanCare SPL/UDL: incoming PDFs were saved although no processing was activated

 

Version 9.10.615 - 29/11/2024

New features

  • Support for Drivve | DM version 11.

 

Version 9.10.605 - 24/10/2024

New Features

  • Add PDF/A 1a and PDF/A 1b in the conversion options.

Error Corrections

  • Sharp Luna Series - Fixed and issue that Luna only see the banner and a white background.

  • When the SharePoint library was changed on the device, it was only changed in the display. As a result, the document was stored in the configured library. Now the selection on the device is applied correctly and the document is stored in the selected folder.

Version 9.10.565 - 02/09/2024

New Features

  • Update Drivve DM components within ScanCare.

Error Corrections

  • Duplicated profile did not have the scan settings from the original profile.

  • Fixed an Issue that, ScanCare lost their licenses.

  • XML data source was not synchronized.

Version 9.10.525 - 18/03/2024

Corrections

• Sharepoint managed meta data input field did not work.

• When adding a device manually, the server URL was not set and it was not possible to press the OK button.

 

Version 9.10.505 - 23/11/2023

Information

  • PDF versions under 1.5 is not supported anymore.

Corrections

  • Single page format produced an error in DefaultDistributor log file.

  • Teams output did not work, when ScanCare uploaded the scanned document to Teams.

  • Sharp OSA3 devices could not connect to ScanCare

  • Toshiba devices scanned only one page if greyscale scan was used.

Version 9.10.385 - 18/01/2023

Enhancements

Version 9.10.275 - 10/08/2022

Enhancements

  • Added support for Azure App and GraphApi Email distributor. New data source available: Office 365/Azure

  • Barcode filter provide new options: "Barcode does not start with", "Barcode does not end with".

  • Restricting the use of the simulator due to security purposes.

  • Lexmark ESF3: Stabilized deployment on older Lexmark devices.

  • Improved SMTP Email output.

 

Version 9.8.265 - May 2022

Enhancements

  • HP - Changed MFP default compression scanner settings to "high".

  • Simulator is now available, if there are only Watchdoc ScanCare desktop clients registered.

  • FUJITSU - Second query based on results of an initial search work on supported FUJITSU devices.

Errors corrections

  • Improved quality of images after conversion with IRIS.

  • Service folder, which was out of the service could not be deleted.

  • Watchdoc ScanCare converted greyscale PDF to monochrome PDF.

  • AD users and groups were not show up correctly on the permissions window.

  • An error message is now shown, when choosing a One Drive folder with too many subfolders.

Version 9.8.240 - March 2022

New features

  • Added Regular expression as filter option to the Barcode separation.

  • Added the option to access the logged in user groups information.

  • Added File Size as a variable.

Enhancements

  • Added function 'Reverse Order' in all tree fields.

  • Added an option for changeable values for margin within the blank page deletion.

  • Added 'PDF rendering with increased DPI' within barcode recognition settings.

  • Support Print from Microsoft Office 365 / Azure OneDrive and Teams.

  • Added new scan setting for Konica Minolta devices 'Mixed original'.

Error Correction

  • Exported documents using IRIS format resulted in inverted colors.

  • Text Functions: Function closing brackets were ignored.

  • In some cases specific barcodes were not recognized.

  • AD users and groups did not show up correctly on the permissions window.

  • Admin app: Advanced conversion settings for pdf did not show up with a new conversion profile.

Version 9.6.485 - March 2022

Enhancements

  • Watchdoc ScanCare can connect to Drivve DM via HTTPS(SSL).

  • Filename is used, when importing a data source.

  • Add further information to the Xerox template selection window.

Corrections

  • Fixed an unhandled error, when adding the Watchdoc ScanCare app to a Xerox device.

  • In some cases, it was not possible to register a new customer on the license server.

  • In specific circumstances created folders, which were required for the scan processes in the Work/Service folder could not be deleted and prevented further actions.

Version 9.6.450 - January 2022

Enhancements

  • Change timeout settings within the DM Output.

Error Corrections

  • UDL showed wrong value for available page.

  • In some cases, Root Folder selection into Drivve Image via Azure (OneDrive/Teams) did not work with a 2016 Server.

  • Print Output did not print all incoming files on UDL profile.

  • Fixed that 'automatically merge pages based on barcode' did not work as expected.

  • In some circumstances, append function created an error message within the image processor log.

Version 9.6.370 - November 2021

New Features

  • Hotfolder with meta file input: ScanCare now ignores the input file until the meta file is available.

  • ScanCare supports now the new Ricoh devices.

Enhancements

  • The online update folder is now cleaned up, after a successful update.

Error Corrections

  • In some cases, ScanCare skipped records in the CSV output file.

  • ScanCare did not save settings in device list if all devices were disabled.

  • In some circumstances ScanCare inserted the value of the mail subject input field several times in the subject depending on the number of attachments.

  • Fixed that, the Job processor was crashing when converting to Word File format and using IRIS.

  • In some cases, if "Export (Password)" was unchecked when exporting Datasource an error occurred.

Version 9.2.205 - May 2021

New Features

Enhancements

  • CustomConfig - In a profile, it is now possible to change properties for more then one Device for the same Type in the CustomConfig.xml

  • Konica Minolta - Added the option to set a value for background removal within the scanner preferences

  • Added support for IATA 2 of 5 barcode

Error Corrections

  • In certain cases the IIS Config tool show success message, even though an error message appears

  • Barcode separation behaves differently with regular barcode recognition and expected barcodes per page feature activated

  • In some cases the Variables and Functions window is blank in the ROUTE output

  • Fixed the Universal data source test function

  • In individual cases {FAX_TO} breaks when <ADS:AccountName> was populated

  • In certain circumstances the ScanCare online update did not stop ScanCare Jobs / ScanCare Conversion service during the update

  • On occasion that IRIS Word conversion lead to messed up document

  • Implemented Heart-Beat monitoring for Watchdoc ScanCare®Service

  • UDL Device: Optimised visibility for the Test button and can be added as DEMO device

  • Watchdoc ScanCare® couldn't read QR Barcodes of certain images
