Systèmes d'exploitations supportés

    • MS® Server 2025 toutes éditions ;

    • Ms® Server 2022 toutes éditions ;

    • MS®Windows 2019 toutes éditions ;

Autres composants

  • Fonctionnalités Windows installées pendant la configuration (un redémarrage du système peut être nécessaire) :

  • Rôle Internet Information Service (IIS) ;

  • Microsoft .NET Framework 8.0 ou version ultérieure ;

  • Espace de stockage disponible : au moins 1,5 Go (données du programme) ;

  • Protocole réseau TCP/IP ;

  • Connexion Internet (pour l'enregistrement du logiciel) ;

  • Configuration minimale requise pour la mémoire vive (RAM) : 4 Go. En fonction du volume et de la structure de vos documents, nous recommandons 8 Go ou plus de mémoire vive (RAM) ;

  • Nous recommandons d'utiliser ScanCare sur des serveurs disposant d'au moins 4 cœurs.
    Avec jusqu'à 6 cœurs, il est recommandé de réserver au moins 1 cœur pour le système d'exploitation :
    Avec 8 cœurs ou plus, nous recommandons de réserver 2 processeurs au système d'exploitation.

    Si d'autres applications serveur que ScanCare sont exécutées sur le même serveur, le nombre de cœurs à leur réserver doit être déterminé en fonction de ces applications. L'utilisation prévue des processeurs et le temps de réponse attendu de ces applications sont deux facteurs à prendre en compte.
    Pour permettre le traitement parallèle des tâches de numérisation, au moins un cœur doit être disponible pour ScanCare pour chaque processus parallèle.

    En fonction du type de documents, il peut être utile de disposer de plusieurs cœurs par tâche de numérisation. Le traitement d'images, par exemple l'OCR et l'analyse des codes-barres, se prête particulièrement bien au traitement parallèle page par page. Cela signifie que si le stock de documents se compose généralement de documents d'une seule page qui sont numérisés individuellement dès leur arrivée, les cœurs supplémentaires pour le traitement ultérieur par tâche de numérisation parallèle n'ont aucun effet sur le temps de traitement.

    Pour les documents qui comportent généralement plusieurs pages ou qui sont numérisés par lots, les cœurs supplémentaires disponibles par tâche de numérisation ont un effet positif sur la vitesse de traitement, en particulier pendant le traitement.

  • Nous vous recommandons vivement d'attribuer une adresse IP fixe au serveur ScanCare.

Surveillance automatique de la mémoire du disque dur

Pendant la numérisation, l'espace libre du disque dur est automatiquement surveillé : si le disque dur manque de mémoire et qu'il ne reste plus qu'1.5, 1 ou 0.5 Go, Watchdoc ScanCare envoie une alerte.

S'il ne reste que 500 Mb de mémoire sur le disque dur, le service s'arrête. Pour redémarrer le service, il est nécessaire de disposer au minimum de 1,5 Gb de mémoire libre sur le disque dur.

N.B. : pour permettre l'envoi d'alertes par e-mail, il convient de configurer cette fonction dans Watchdoc.

Serveurs

Compatibilité possible vers les applications cibles suivantes :

  • Microsoft® Office SharePoint* Server 2007/2010 MOSS Standard

  • Microsoft® Office SharePoint* Server 2007/2010 MOSS Enterprise

  • Microsoft® SharePoint* 2010

  • Microsoft® SharePoint* Foundation 2010

  • Microsoft® SharePoint* 2013 (toutes éditions 2013)

  • Microsoft® Office 365 (l'authentification basique (Basic) doit être activée)

* Aucun logiciel supplémentaire ne doit être installé sur le serveur SharePoint. Les protocoles HTTP et HTTPS sont supportés.

Serveurs E-Mails supportés

  • Microsoft® Exchange* 2016 (utilisant Exchange Web Services)

  • Microsoft® Exchange* 2013 (utilisant Exchange Web Services)

  • Microsoft® Exchange* 2010 (utilisant Exchange Web Services)

  • Microsoft® Office 365

  • tout server mail compatible SMTP (SSL supporté)

  • Lotus® Domino (Version 4.4 et supérieures)

* Aucun logiciel ou client e-mail (tel que MAPI MFP signifie 'Multi Fonction Peripheral' et désigne les périphériques offrant, outre la fonction d'impression, celle de copie et de numérisation.) ne doit être installé sur le serveur MS®Exchange. HTTP et HTTPS sont supportés. En fonction de la version MS®Exchange, WebDAV ou Exchange Web Services doit être activé sur le serveur MS® Exchange.

Serveurs e-mails non supportés :

  • Novell Groupwise

Prérequis organisationnels

Avant d'entamer la procédure d'installation et de configuration, il convient de vérifier les prérequis suivants :

  • au minimum, un périphérique d'impression est installé sur le serveur ;

  • vous disposez d'une licence acquise auprès du service Commercial de Doxense® ;

  • le service IIS est installé sur le serveur hébergeant la solution ScanCare ;

  • pour paramétrer les notifications par mail, vous disposez des informations relatives au serveur de messagerie (nom du serveur, n° de port, informations d'authentification, etc.) ;

  • vous disposez d'une connexion internet opérationnelle (pour l'étape d'enregistrement).

  • l'installation de ScanCare sur des contrôleurs de domaine ou des serveurs de terminaux n'est explicitement pas recommandée. Doxense ne prend pas en charge la résolution des problèmes liés à une telle installation.

La fonction d'OCR nécessite une capacité importante en CPU. C'est pourquoi nous vous recommandons d'installer Watchdoc ScanCare sur un serveur physique plutôt que sur un serveur virtuel.
Nous vous recommandons également d'utiliser l'outil OCR-Benchmark tool (dans le dossier d'installation ScanCare) pour vérifier la vitesse du serveur. Vous pourrez ainsi estimer le temps nécessaire à l'OCR et les capacités serveur nécessaires à cette fonction.
Le redémarrage de l'ordinateur peut être nécessaire après l'installation de Watchdoc ScanCare.

Droits et permissions requis pour le compte de service ScanCare

L'installation de Watchdoc ScanCare nécessite de disposer d'un compte Administrateur.

Afin d'éviter d'avoir à modifier les paramètres du compte Administrateur existant, nous vous recommandons de créer un compte de service spécifique à l'application Watchdoc ScanCare, inclus dans le groupe local des administrateurs du domaine.

Pour installer ScanCare, vous devez être connecté à MSWindows en utilisant le compte de service ScanCare.

 

Expression Signification Dossier par défaut
<InstallRoot> Dossier d'installation ScanCare C:\Program files\Doxense\ScanCare
<CommonFilesDir> Fichiers communs de MS Windows C:\Program files\Common files
<WinDir> Dossier de MS Windows C:\Windows

 

Les actions suivantes doivent être effectuées sur le compte d'utilisateur sélectionné :

  • ajoutez le compte au groupe local Administrateurs

  • donnez un accès complet aux dossiers : <InstallRoot>\Hotfolder, <InstallRoot>\HotfolderPrint, <InstallRoot>\Temp, <InstallRoot>\Log, <InstallRoot>\DoxenseWSWorkFolder, <CommonFilesDir>\Doxense, <WinDir>\Temp

  • donnez un accès en lecture à <InstallRoot>, y compris tous les sous-dossiers.

 

Permissions à accorder au compte utilisateur sélectionné :

  • se connecter en tant que service

  • agir en tant que partie du système d'exploitation

  • se connecter en tant que travail par lot

  • usurper l'identité d'un client après authentification

  • augmenter le quota de mémoire pour un processus

  • attribuer un jeton primaire

  • remplacer un jeton de niveau processus

  • prendre la propriété de fichiers ou d'autres objets

 

Droits sur les dossiers :

  • Accès complet pour le groupe des utilisateurs locaux et le groupe des utilisateurs du domaine (le cas échéant) : <InstallRoot>\Temp, <InstallRoot>\Log, <InstallRoot>\DoxenseWSWorkFolder, <CommonFilesDir>\Doxense
